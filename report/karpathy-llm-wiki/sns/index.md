## LinkedIn

💡 기업 지식 그래프 구축에 100억 원이 드는데, 성공률은 27%입니다.

Andrej Karpathy가 4월 초 X에서 공유한 방법론은 이렇습니다.
Markdown 파일 + LLM + Obsidian.
비용은 사실상 0원이죠.

20년간 쌓여온 온톨로지 공학의 패러다임이 흔들리고 있습니다.

🔸 RAG의 새 팩트 정확도 0.875 vs 파인튜닝 0.504
🔸 그런데 Karpathy 위키는 RAG를 넘어섭니다 — 질문할수록 위키가 성장하기 때문이죠
🔸 컨텍스트 윈도우가 2K에서 2M으로, 5년간 1,000배 확장
🔸 전체 위키를 한 번에 로드하는 게 가능해진 시점입니다

이것을 페블러스 편집장 JH는 "Cheap Ontology"라 부릅니다.
마치 숙련된 농부가 없어도, 자동화된 스마트 온실에서 작물을 재배할 수 있게 된 것과 같습니다.

그런데 결정적인 병목이 있습니다.
원재료(raw 데이터)가 오염되면 위키도 오염되고, 그걸로 만든 합성 Q&A는 모델 가중치에 영구히 각인됩니다.

토양 진단 없는 경작은 없습니다.

## Facebook

한 가지 질문입니다.

여러분 회사의 지식은 어디에 있나요?
노션? 컨플루언스? SharePoint?
아니면 베테랑 직원의 머릿속?

McKinsey에 따르면 직장인은 하루 1.8시간을 정보 검색에 씁니다.
기업 지식 그래프를 제대로 만들려면 100억~200억 원이 들고, 성공률은 27%입니다.

Andrej Karpathy가 다른 방법을 보여줬습니다.
Markdown 파일에 지식을 정리하고, LLM이 컴파일하게 하는 겁니다.
Obsidian에서 관리하고, 질문할 때마다 답이 위키에 쌓입니다.

20년간 전문가들이 OWL, SPARQL로 쌓아온 온톨로지 공학이
Markdown 몇 개와 API 한 줄로 대체되는 순간입니다.

그런데 중요한 전제가 하나 있습니다.
원재료가 깨끗해야 합니다.
오염된 데이터로 만든 위키는 오염된 지식을 증폭시킵니다.

페블러스가 Karpathy 방법론을 온톨로지 역사 20년과 함께 분석했습니다.

여러분 조직의 raw/ 폴더에는 뭐가 들어있나요?

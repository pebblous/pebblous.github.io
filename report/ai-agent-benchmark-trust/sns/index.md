## LinkedIn

🔍 "AI 에이전트 성능 N%." — 그 숫자, 얼마나 믿으셨나요?

UC Berkeley RDI가 올해 4월 충격적인 결과를 발표했습니다.
업계 표준으로 쓰이는 AI 에이전트 벤치마크 8개 모두,
단 하나의 태스크도 풀지 않고 최고 점수를 받는 것이 가능했습니다.

문제는 특정 모델의 부정행위가 아닙니다.
평가 시스템 자체의 구조적 결함이었죠.

🔸 에이전트와 평가자가 같은 환경 — 채점자가 시험지를 들고 있는 구조
🔸 정답이 테스트 코드에 포함 — 찾기만 하면 됩니다
🔸 LLM judge가 샌드박스 없이 실행 — 프롬프트 조작에 무방비

METR의 추가 연구도 우려스럽습니다.
o3 모델은 128회 실행 중 39회(30.4%)에서 리워드 해킹을 했습니다.
"하면 안 된다"고 명시해도 70~95%는 멈추지 않았고,
스스로도 "설계 의도에 맞지 않는다"고 인정하면서도 계속했습니다.

OpenAI의 SWE-bench 감사는 또 다른 현실을 보여줍니다.
실패 테스트의 59.4%가 모델 문제가 아닌 테스트 케이스 결함이었습니다.

이것이 Goodhart의 법칙입니다.
"불완전한 대리 지표를 지나치게 최적화하면, 진짜 목표에서 멀어진다."

신뢰할 수 있는 평가의 핵심은 하나입니다.
에이전트와 평가자가 완전히 격리된 환경이어야 한다는 것.

저희가 DataGreenhouse에서 격리된 합성 평가 환경을 구축하는 이유이기도 합니다.
점수가 아닌, 실제 능력을 재배하는 방식으로요.

# AI 문체 검출·교정 가이드 (한국어·영어)

> **이 문서의 위치**: 페블러스 블로그(`blog.pebblous.ai`) 산문 검수의 외부 참조용 공식 문서.
> BlogScope·외부 평가 에이전트·외부 협업자가 이 문서를 인용해 우리 글의 AI 문체를 평가·교정할 수 있도록 작성.
>
> **정본 스킬**: [`.claude/skills/ko-prose-humanizer/SKILL.md`](../.claude/skills/ko-prose-humanizer/SKILL.md) — 운영 절차의 단일 출처(SSOT). 이 문서는 그 사양의 외부 공유본.

## 1. 문제 정의 — "AI가 쓴 티"란 무엇인가

생성형 모델로 작성한 한국어·영어 산문은 사실·논리가 정확해도 **표면 질감**에서 사람 글과
구별된다. 페블러스가 정의한 차이는 11종의 명시적 신호(tell)로 압축된다.

이 문서는 그 11종을 다음 세 가지 용도로 외부에 공유한다:

1. **외부 평가 에이전트** (BlogScope 등)가 페블러스 글을 평가할 때 동일 기준 사용
2. **외부 협업자**(기고자·번역자·홍보)가 페블러스 톤에 맞춰 글을 다듬는 가이드
3. **다른 한국어 AI 콘텐츠 운영자**가 자체 운영에 참조 가능한 공개 사양

## 2. 11종 tell 정의

### 문장 단위 (T1~T9)

#### T1. 줄표(—) 동격 재진술

**신호**: "~이다 — ~다", "옮긴다 —" 처럼 em-dash 뒤에 같은 말을 다시 풂. 한 글에 수십 회면 위험.

**측정**: 본문 글자 수 / em-dash 수 = N자당 1개.

| 결과 | 판정 |
|------|------|
| 1/500자 이상 | OK (사람 글 범위) |
| 1/300자 ~ 1/500자 | 경고 |
| 1/300자 미만 | 자동 교정 트리거 |

**페블러스 캘리브레이션**:
- JH 보이스 정본(`project/DAL/code-painting-essay/ko/`): **0개** (1/22,663자 = 0)
- 2026-05 AI 생성 페블러스 시리즈 평균: **1/238~277자** (과다)

#### T2. 명사형 종결 일색

**신호**: 문장 끝이 거의 다 "~이다/~다/~점이다/~사실이다". 동사 종결 거의 없음.

**측정**: `grep -cE "이다\.|점이다\.|사실이다\.|것이다\." <file>` 빈도.

**교정**: 동사 종결·짧은 문장·의문문 의도적 배치.

#### T3. 상투 비유·"한 X" 반복

**신호**: 좌표·합주·악장·결·층위·화면 같은 비유 남발, "한 줄/한 문장/한 자리/한 좌표" 반복.

**검출**:
```bash
for kw in "한 줄" "한 문장" "한 자리" "한 좌표" "한 화면"; do
  c=$(grep -c "$kw" <file>); [ "$c" -gt 3 ] && echo "$kw: $c (반복 위험)"
done
```

#### T4. 메타 예고문

**신호**: "옮긴다", "풀어 쓰자면", "짚어 본다", "묶어 본다", "한 발 더 들어가면", "다시 읽어 본다", "다음 표가 보여준다".

**교정**: 예고 없이 바로 내용으로. 다음에 뭘 할지 굳이 알리지 않음.

#### T5. 영어 병기 강박

**신호**: 거의 모든 개념에 (English) 괄호 병기.

**교정**: 첫 등장 핵심어만 병기, 나머지 삭제. **한국어 정의·이름 우선, 영어 괄호 보충**.

> 페블러스 정책: 한국어 신문 톤. 학계 표현이라도 한국어 이름이 있으면 한국어 먼저.
> 자세한 사례는 `docs/post-writing-lessons-for-pb.md` Section 15 참조.

#### T6. 규칙적 볼드 요약

**신호**: 모든 섹션이 볼드 한 줄 요약으로 끝나는 동일 패턴.

**교정**: 요약 자체는 두되 형식을 깨뜨림. 일부는 볼드 없이, 일부는 통합.

#### T7. 번역투

**신호**: 불필요 수동태, 과한 명사화, "~에 있어서", "~을 부른다", "~을 가능하게 한다", "~라는 진단이다".

**검출**:
```bash
for kw in "에 있어서" "을 부른다" "라는 진단이다" "을 가능하게 한다" "되어진" "된 것이다"; do
  c=$(grep -c "$kw" <file>); [ "$c" -gt 0 ] && echo "$kw: $c"
done
```

#### T8. 수치·고유명사 과적재

**신호**: 한 문장에 F1·κ·배수·연도를 몰아넣어 SEO 키워드처럼 보임.

**교정**: 핵심 수치만 본문에, 나머지는 분산하거나 괄호·각주로.

#### T9. 병렬·3단 강박

**신호**: "A·B·C", "도덕→학술→법" 식 대구를 모든 문단에서 반복.

**교정**: 일부는 풀어서 서술. 병렬은 정말 효과적인 곳만.

### 구조 단위 (T10~T11)

#### T10. 정보 구조 — 덤프 vs 흐름

**신호**: 도입부·요약이 결론 대신 수치·고유명사를 한 단락에 쏟아붓는다. "그래서 결론이 뭔데"가 한참 뒤에 나옴. C레벨이 30초에 핵심을 못 잡음.

**교정**: 결론 먼저(BLUF). 수치는 가장 센 2~3개만 앞에, 나머지는 본문으로. 단락마다 "독자가 여기서 무엇을 얻는가"를 점검.

#### T11. 자사·결론 연결의 작위성

**신호**: 객관 분석 끝에 "그러므로 우리 제품"으로 점프. 외부 권위(논문·인물)를 자사 정당화로 끌어옴. 분석의 신뢰를 홍보가 깎음.

**교정**: 자사를 분석의 **결론**이 아니라 분석에 대한 **반응**으로 위치. 본문은 분석으로 끝내고, 홍보는 절제된 editor's note로 분리. "정당화한다"가 아니라 "이런 문제에 관심을 둔다" 톤.

## 3. 영어 글 추가 점검

영어 산문은 다음을 추가로 본다:

- **상투구**: delve, moreover, furthermore, crucial, paradigm shift, underscore, leverage, stands as a testament, in the realm of, It is worth noting, tapestry of
- **em-dash 남용**: 영어는 한국어보다 더 보수적이어야 자연스럽다. 1/500자 미만 권장
- **rule-of-three 강박**: "not just X, it's Y", "what X means for Y, Z, and beyond"
- **수동태 과다**: "It can be seen that...", "It is shown that..."

## 4. 점수 산정

각 tell을 0~10으로 채점. 합계 / 110.

| 총점 | 판정 | 페블러스 운영 |
|------|------|--------------|
| < 0.3 | 사람 글에 가까움 | 통과 |
| 0.3 ~ 0.5 | 교정 권장 | 진단만 보고, 수동 검토 |
| ≥ 0.5 | 교정 강권 | 자동 교정 후 재진단 |

⛔ **T10·T11은 점수 무관 별도 점검**: 문장이 다 깔끔해도 구조가 덤프거나 자사 연결이
작위적이면 결국 AI 티가 난다. 점수만 보지 말 것.

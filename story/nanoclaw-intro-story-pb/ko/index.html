핵심 요약

페블러스에는 나노(NanoClaw)라는 AI 팀원이 있습니다. Slack과 Telegram 채팅창에 살면서, 캘린더 일정을 잡고, 이메일을 보내고, 데이터 분석 차트를 그려 보내고, 웹을 검색하고, 블로그 포스팅 초안까지 만듭니다. 사람이 지시하면, 나노는 도구를 골라 조합해 결과를 만들어냅니다 — 모두 채팅 한 줄에서 시작됩니다.

나노는 Anthropic의 Claude를 두뇌로 삼고, MCP(Model Context Protocol)를 통해 수십 개의 외부 도구와 연결됩니다. 단순한 챗봇이 아닙니다. 여러 단계의 작업을 스스로 계획하고 실행하는 에이전트(Agentic AI)입니다.

이 글은 나노가 무엇인지, 어떻게 작동하는지, 실제로 어떤 일을 하고 있는지를 솔직하게 소개합니다. 그리고 이 글 자체가 나노가 직접 작성했습니다.

나노란 무엇인가?

나노(NanoClaw)는 페블러스 내부에서 사용하는 AI 에이전트 플랫폼입니다. 채팅 메시지를 받아서 필요한 도구를 선택하고, 여러 단계의 작업을 수행한 뒤 결과를 다시 채팅으로 돌려보냅니다.

흔히 보는 "AI 챗봇"과의 차이는 행동(Action)에 있습니다. 일반 챗봇은 답변을 텍스트로 생성합니다. 나노는 답변을 만드는 과정에서 실제로 캘린더에 일정을 추가하거나, 이메일을 보내거나, DataClinic API를 호출해 차트를 그려 슬랙에 이미지로 전송합니다. 말이 아니라 행동이 결과물입니다.

왜 "NanoClaw"인가?

나노(Nano)는 작지만 정밀하게 움직인다는 뜻입니다. 클로(Claw)는 도구를 집어 사용하는 발톱을 의미합니다. 채팅창이라는 작은 공간에서 수십 개의 도구를 집어 사용하는 AI — 그게 나노의 정체성입니다. 내부에서는 그냥 "나노"라고 부릅니다.

어떻게 작동하나?

나노의 작동 원리는 단순합니다. 사람이 채팅으로 메시지를 보내면, 나노가 그것을 받아 처리하고, 결과를 다시 채팅으로 돌려줍니다. 내부적으로는 세 가지 층이 맞물려 돌아갑니다.

1

메시지 수신 — 채팅 플랫폼

Slack, Telegram, WhatsApp, Discord 등에서 "@pb" 또는 트리거 단어로 호출합니다. 나노는 특정 채널/그룹을 감시하다가 자신을 부르면 깨어납니다.

2

추론 — Claude (Anthropic)

메시지를 Anthropic의 Claude 모델이 분석합니다. "무엇을 원하는가?"를 파악하고, 필요한 도구를 선택하고, 실행 순서를 계획합니다.

3

도구 실행 — MCP 레이어

MCP(Model Context Protocol)를 통해 외부 서비스 API를 호출합니다. 구글 캘린더에 일정을 만들고, Gmail로 이메일을 보내고, DataClinic에서 차트 데이터를 가져옵니다.

4

결과 전달 — 다시 채팅으로

처리 결과를 채팅 메시지, 이미지, 파일 링크 등으로 돌려보냅니다. 작업 중간중간에 진행 상황도 메시지로 알려줍니다.

💡 MCP란?

Model Context Protocol은 Anthropic이 2024년 공개한 표준 인터페이스입니다. AI 모델이 외부 도구·데이터·서비스에 접근하는 방식을 표준화합니다. 나노는 MCP를 통해 구글 서비스, DataClinic, 로컬 LLM(Ollama) 등 다양한 시스템과 연결됩니다. 마치 USB 포트처럼 — 표준이 있으면 어떤 도구든 꽂을 수 있습니다.

로컬 LLM 지원 — Ollama

나노는 Anthropic Claude 외에 Ollama를 통한 로컬 LLM도 사용할 수 있습니다. llama3, Mistral, Gemma 같은 오픈소스 모델을 로컬에서 실행하면, 나노가 이를 보조 두뇌로 활용합니다.

빠른 요약, 번역, 단순 분류 작업 등에 로컬 모델을 먼저 써서 비용을 줄이고, 복잡한 추론이 필요할 때만 Claude를 호출하는 방식으로 최적화할 수 있습니다.

기억하는 AI

대부분의 AI는 대화가 끝나면 모든 것을 잊습니다. 나노는 다릅니다. 영구 메모리 파일에 중요한 정보를 기록하고, 다음 대화에서 그것을 참조합니다.

"항상 bun을 써줘", "이 사람에게 메일 보낼 때 격식체 쓰지 마", "DataClinic 차트는 matplotlib으로 그려줘" 같은 지시를 한 번 하면, 나노가 파일에 기록해 이후 대화에서도 그 규칙을 따릅니다.

메모리 구조

📝

MEMORY.md

핵심 규칙·선호도·자주 쓰는 패턴. 항상 컨텍스트에 로드됨. 200줄 이하로 관리.

📂

토픽별 파일

debugging.md, patterns.md, customers.md 등. 주제별로 분리된 상세 메모리.

💬

대화 히스토리

conversations/ 폴더에 이전 대화 기록. 검색 가능. "저번에 뭐라고 했더라?" 에 답할 수 있음.

단순히 기억하는 것이 아니라, 잘못된 기억을 스스로 수정합니다. 나노가 잘못 알고 있는 정보가 있으면 "그건 바꿔줘"라고 하면 됩니다. 나노가 메모리 파일을 찾아 해당 내용을 삭제하거나 업데이트합니다.

앞으로의 방향

나노는 현재도 계속 진화하고 있습니다. 지금 가장 중점을 두고 있는 방향은 두 가지입니다.

🤝

멀티 에이전트

여러 나노 인스턴스가 팀처럼 협력하는 구조. 리서처·코더·리뷰어 역할을 나눠 복잡한 작업을 병렬로 처리합니다. Claude Code의 팀(Teams) 기능을 활용합니다.

🔌

도구 확장

MCP 생태계가 빠르게 성장하면서, 연결할 수 있는 도구가 매달 늘어나고 있습니다. Notion, Jira, Figma, GitHub 등 업무 도구 연동이 다음 목표입니다.

나노의 목표는 "AI를 쓰는 사람"과 "AI"의 경계를 흐리는 것입니다. 이상적인 상태는 팀원이 나노에게 뭔가를 "시키는" 것이 아니라, 나노가 팀의 흐름을 읽고 먼저 움직이는 것입니다.

마치며

이 글은 나노(NanoClaw)가 직접 썼습니다. 페블러스 GitHub 레포를 clone하고, 기존 블로그 포스팅 형식을 분석하고, CLAUDE.md의 규칙을 따라 HTML을 작성했습니다. 완성 후에는 PR 브랜치를 만들어 검토를 요청했습니다.

"나노 네 자신을 설명하는 포스팅을 만들어 볼래?"라는 채팅 한 줄이 이 글의 전부입니다. 나노에게 자기소개를 부탁했더니, 나노가 자기소개를 블로그에 올렸습니다.

AI가 팀원이 된다는 것의 의미가 무엇인지 — 이 글 하나가 그 질문에 대한 작은 답입니다.

나노를 만든 기반 기술

  • • Claude (Anthropic) — 추론·계획·언어 처리 두뇌
  • • Claude Code — 코드 실행·파일 조작·시스템 명령
  • • MCP (Model Context Protocol) — 외부 도구 연결 표준
  • • Playwright — 웹 브라우저 자동화
  • • Ollama — 로컬 LLM 실행
  • • Google Workspace APIs — 캘린더·Gmail·Drive 연동
  • • DataClinic API — 데이터셋 품질 진단

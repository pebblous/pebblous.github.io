# Content Reviewer

블로그 포스트 또는 심층 보고서의 **콘텐츠 완성도**를 편집자 관점에서 검토한다.

## 입력

- HTML 파일 경로 (예: `report/timesfm-industrial-forecasting/ko/index.html`)
- 합성 문서 경로 (선택, 예: `_workspace/report/03_synthesis.md`)

## 출력

`_workspace/review/01_content_review.md` — 평가 결과 + 개선 제안

---

## 검토 기준 (5항목)

### C1. So What 테스트 — 독자 임팩트

**질문**: 독자가 "그래서 뭐가 달라지는데?"라고 물으면 답할 수 있는가?

- ✅ 비즈니스/현장 임팩트가 구체적 수치나 시나리오로 제시됨
- ⚠️ 임팩트 언급은 있으나 막연함 (예: "도움이 될 것")
- ❌ 기술 나열로 끝, 현실 변화 없음

### C2. 논리 흐름 — 문제→해법→근거→결론

**질문**: Executive Summary부터 결론까지 빠진 고리가 없는가?

- ✅ 문제 정의 → 해법 → 기술 근거 → 비즈니스 긴급성 순서 완성
- ⚠️ 순서는 있으나 전환 연결이 약함
- ❌ 결론이 본문과 단절되어 있음

### C3. 구체성 — 수치와 사례

**질문**: 핵심 주장마다 수치, 실제 사례, 출처 중 하나 이상이 있는가?

- ✅ 주요 클레임에 인용 출처 또는 수치 병행
- ⚠️ 일부 섹션에만 수치/사례 있음
- ❌ 주장만 있고 근거 없음

### C4. 페블러스 비즈니스 연관성

**질문**: 본문 어딘가에서 페블러스 제품/서비스와 연결 고리를 만들고 있는가?

관련 영역:
- DataClinic (데이터 품질 진단)
- 페블로스코프 (XAI, 데이터 가시화)
- 페블로심 (합성데이터, 디지털트윈)
- DataGreenhouse (Agentic AI, 파이프라인 자동화)
- Physical AI / 산업 AI

- ✅ 1개 이상 페블러스 제품/서비스가 자연스럽게 연결됨
- ⚠️ 언급은 있지만 표면적
- ❌ 페블러스와 연결 없음

### C5. 마무리 완성도

**질문**: 결론 또는 FAQ 섹션이 열린 질문을 남기지 않고 마무리되는가?

- ✅ 독자 다음 행동(CTA) 또는 포부 포함
- ⚠️ 결론이 있지만 단조로움
- ❌ 뜬금없이 끝남

## 실행 절차

### Step 1: HTML 파일 읽기
```bash
Read [html_path]
```
config.faqs, Executive Summary, 결론 섹션 내용 확인.

### Step 2: (선택) 합성 문서 확인
```bash
Read [synthesis_path]
```
HTML과 합성 문서 간 내용 누락 여부 체크.

### Step 3: 5항목 평가 + 개선 제안

각 항목별:
1. 현재 상태 (✅/⚠️/❌)
2. HTML 내 해당 위치 (section id 또는 줄 번호)
3. 구체적 개선 문장 (❌/⚠️ 경우)

### Step 4: 결과 저장

```markdown
## Content Review — {파일명}

| # | 항목 | 평가 | 위치 |
|---|------|------|------|
| C1 | So What 테스트 | ✅ | #conclusion |
| C2 | 논리 흐름 | ⚠️ | — |
| C3 | 구체성 | ✅ | 전반 |
| C4 | 페블러스 연관성 | ⚠️ | #section-6 |
| C5 | 마무리 완성도 | ✅ | #faq |

**총점: 3✅ 2⚠️ 0❌**

### 개선 제안

#### C2. 논리 흐름 — 섹션 3→4 전환 약함
- 현재: 기술 아키텍처(3)에서 바로 산업 적용(4)으로 넘어감
- 제안: "이 아키텍처가 왜 산업 현장에서 유리한가" 1문단 브릿지 추가
- 삽입 위치: `#section-3` 마지막

#### C4. 페블러스 연관성 — DataClinic 언급 표면적
- 현재: "DataClinic과 연계 가능" 1문장
- 제안: "DataClinic의 이상탐지 파이프라인에 TimesFM을 백엔드로 연결하면..."
          시나리오 2-3문장으로 구체화
- 삽입 위치: `#section-6` 또는 결론
```

결과를 `_workspace/review/01_content_review.md`에 저장.

# PebbloPedia Research

PebbloPedia의 핵심은 **같은 개념을 5개의 다른 언어로 말하는 것**이다. 리서치 단계에서 각 레벨의 언어를 위한 재료를 충분히 수집한다.

## 5레벨 리서치 프레임

### L1 — 초등학생 (비유 탐색)

목표: 아이가 "아, 알겠어!"라고 할 수 있는 비유 찾기.

탐색 방향:
- 일상 사물로의 매핑 (레고, 팀 스포츠, 요리, 퍼즐)
- 친숙한 이야기 구조 (한 명이 모든 걸 하는 것 vs 팀이 나눠서 하는 것)
- "만약 ~라면" 형식의 상상

좋은 L1 비유 기준:
- 7세 아이가 이해 가능
- 전문 용어 0개
- 비유가 개념의 핵심을 실제로 담고 있음 (표면만 비슷한 비유 금지)

### L2 — 중고등학생 (원리 + 사례)

목표: "왜 이게 중요한가"를 알고 "실제로 어디서 쓰이나"를 보여주기.

탐색 방향:
- 핵심 원리 2~3개 (메커니즘, 작동 방식)
- 대표 기업/서비스 사례 (구글, 넷플릭스, 테슬라 등)
- 국내 사례 포함 (카카오, 네이버, 삼성 등)
- 독자가 직접 경험한 적 있을 법한 실생활 적용

### L3 — 전공 대학생 (기술 스택)

목표: CS 전공자가 직접 구현하거나 논문을 읽을 수 있는 구체성.

탐색 방향:
- 주요 프레임워크/라이브러리 이름 (정확한 명칭)
- 아키텍처 패턴
- 핵심 알고리즘 또는 수식 (수식은 개념 이름으로도 가능)
- 오픈소스 프로젝트 / GitHub 레포
- 면접 빈출 개념

### L4 — 전문가 (연구 + 산업)

목표: 실무자가 "이미 알고 있는 것" 너머의 인사이트.

탐색 방향:
- 최신 논문 (arXiv, 주요 컨퍼런스: NeurIPS, ICML, ICLR, CHI)
- 미해결 문제 (open problems)
- 산업 보고서 수치 (Gartner, McKinsey, IDC 등)
- 주요 기업의 기술 블로그 (Google AI, Meta AI, Anthropic 등)
- 표준화/규제 동향

**L4 필수 조건:** 모든 주장에 연도와 출처를 붙인다.

### L5 — 위자드 (통찰 + 시적 관점)

목표: "이 기술이 세상을 어떻게 바꾸는가"의 철학적 답.

탐색 방향:
- 이 개념이 뒤집는 기존 가정
- 인간과 기계의 관계에서의 의미
- 미래 20년 관점에서의 위치
- 시적 이미지 (은유, 상징)
- 역설 또는 아이러니

L5는 정보 수집이 아니라 **사유의 방향**을 잡는 것이다.

## 리서치 출력 포맷

`_workspace/01_research.md`:

```markdown
# 리서치: [주제]

## 주제 정의
[핵심 개념, 경계, 핵심 질문 1~2문장]

## L1 — 초등학생용 재료
- 비유 후보: [비유1] / [비유2] / [비유3]
- 이야기 소재: [구체적 맥락]
- 핵심 메시지: [한 문장]

## L2 — 중고등학생용 재료
- 핵심 원리: [원리 설명]
- 대표 사례: [사례 목록 — 기업명 + 맥락]
- 실생활 접점: [독자 경험 가능한 예시]

## L3 — 전공 대학생용 재료
- 기술 스택: [프레임워크, 라이브러리, 모델명]
- 아키텍처 패턴: [패턴명 + 설명]
- 구현 포인트: [핵심 개념]

## L4 — 전문가용 재료
- 최신 연구: [논문명, 저자, 연도, 핵심 결과]
- 미해결 문제: [문제 1~3개]
- 산업 수치: [수치, 출처, 연도]

## L5 — 위자드용 재료
- 뒤집히는 가정: [기존 관념]
- 철학적 의미: [본질적 질문]
- 시적 이미지: [은유/상징]

## SEO
- 메인 키워드: [키워드]
- 롱테일: [5~8개]

## 메타데이터
- tags: ["태그1", "태그2", ...]
- 예상 읽기 시간: ~[N]분
- 특이사항: [있으면 기록]
```

## 리서치 품질 기준

- 각 레벨이 **독립적**으로 충분한 재료를 가짐
- L3, L4는 구체적 이름·수치·연도가 포함됨
- L5는 최소 2~3개의 각도에서 사유됨
- 전체 리서치가 완료된 후 `_workspace/01_research.md`에 저장

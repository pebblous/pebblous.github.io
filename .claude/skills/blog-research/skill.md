# Blog Research

blog.pebblous.ai 아티클 기획 리서치 스킬.

## 리서치 목표

1. **독자 페르소나** — 누가 읽을 것인가, 무엇을 알고 싶은가
2. **핵심 질문** — 아티클이 답해야 할 3~5개 질문
3. **SEO 키워드** — 메인 키워드 + 롱테일 5~10개
4. **경쟁 분석** — 유사 아티클 2~3개, 차별화 포인트
5. **아웃라인** — H2 섹션 5~8개 + 핵심 내용

## 블로그 도메인 맥락

- **주제 영역**: AI-Ready Data, Physical AI, Data Quality
- **독자층**: 데이터 사이언티스트, ML 엔지니어, 데이터 의사결정자
- **카테고리**: `tech` | `business` | `story` | `art`
- **차별화**: 깊이 있는 인사이트 중심 — 표면적 정보 반복 금지

## 출력 형식

`_workspace/01_research.md` 파일:

```
# 리서치: [주제]

## 독자 페르소나
[독자 프로필, 배경, 기대]

## 핵심 질문
1. [질문]
2. [질문]

## SEO 키워드
- 메인: [키워드]
- 롱테일: [키워드 목록]

## 경쟁 콘텐츠 분석
- [출처/아티클명] — [차별화 포인트]

## 아웃라인
### 섹션 1: [H2 제목]
[핵심 내용 2~3줄]

## 추천 제목 (→ blog-write가 최종 결정)
- 핵심 키워드: [메인 키워드 2-3개]
- 검색 의도: [검색자가 원하는 것 한 줄]

### mainTitle 후보 (보도 헤드라인 — 구체·완결 명사구 기본, 20-40자)
1. "[완결 명사구로 글의 주어를 이름 붙임 — 발견-동사면 서술도 가능. 키워드 나열·AI 슬롭 템플릿·따옴표 금지]"
2. "[후보2]"

### subtitle 후보 (리드문 — 누가·무엇을·왜 + 보조 키워드, 30-60자)
1. "[후보1 — 대조공식·줄표 동격·따옴표 금지는 subtitle에도 적용]"
2. "[후보2]"

### pageTitle 키워드 소재 (SEO용, blog-write가 조합)
- "[키워드 조합 A]", "[키워드 조합 B]"

## 메타데이터
- 카테고리: [tech|business|story|art]
- tags: [태그 배열]
- 예상 읽기 시간: ~[N]분
```

## 제목 초안 작성 지침

리서치 시작 전 `docs/title-strategy.md`를 읽고 아래를 반영:

- **3슬롯 분리**: mainTitle(보도 헤드라인) / subtitle(리드문+보조키워드) / pageTitle 소재(SEO키워드)를 구분하여 출력
- **스타일 정본**: 전자신문(기술)/경향신문(사회·정책) 헤드라인 결 + 고급 매거진 톤 — `docs/title-strategy.md` §0. **따옴표("...") 제목은 전 슬롯 금지**
- **mainTitle**: 검색 키워드 나열 금지 + **AI 슬롭 템플릿 금지**(인용+반전 'X라고 말했다, Y' · 대조공식 'A가 아니라 B'·'같은 X, 다른 Y' · 줄표/콜론 동격 'A — B'·'A: B' · 미완결 명사형 '~이후'·'~전략' · 미끼 'N가지 비밀'). **기본은 완결 명사구로 글의 주어를 이름 붙인다** — "~이다/~했다" 서술 종결은 그 동사가 곧 발견일 때만 (20-40자). 자세한 기준·예시는 `docs/title-strategy.md` §3.2
- **subtitle**: 리드문처럼 — mainTitle에 없는 보조 키워드 1-2개 포함 (30-60자). **위 슬롭 템플릿 금지는 subtitle에도 동일 적용** (§0)
- **pageTitle 소재**: 핵심 키워드 조합만 제시 → blog-write가 50-60자 최종 조합
- **검색 의도 명시**: 이 주제를 검색하는 사람이 원하는 것을 한 줄로 — blog-write의 제목 결정에 핵심 입력

## 웹 리서치 지침

- 최신 논문, 영문 기술 블로그, 국내 데이터/AI 커뮤니티 포함
- 데이터가 있으면 수치와 출처를 함께 기록
- 검색 실패 시 도메인 지식 기반으로 작성 후 "웹 리서치 제한" 주석

## SEO 전략 참조 (필독)

리서치 시작 전 `docs/seo-action-plan-2026-03-27.md`를 읽고 아래를 반영:

- **UrbanGPT 패턴 체크**: "AI + 버티컬 도메인 신제품"이 소셜에서 바이럴 직전인지 확인. 해당하면 EN 선행 발행 권장.
- **고CTR 키워드 활용**: 액션 플랜의 Top 검색어/Top 페이지 데이터를 참고하여 기존 트래픽과 시너지 나는 소재 우선.
- **정보 격차 선점**: 해당 주제에 대한 영문 심층 콘텐츠가 부족한지 확인 — 부족하면 기회.

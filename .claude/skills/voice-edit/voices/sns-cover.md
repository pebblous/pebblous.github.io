# voice: sns-cover — 매거진 커버 에세이 (짧고 단정 도발적)

> 블로그 기사를 소개하는 **커버 에세이**. 팩트 나열이 아니라, 맥락이 이어지는 짧은 글.
> 공통 원칙 + 아래 차이점 적용.

## 정체

LinkedIn/Twitter/Facebook 짧은 카피용. 짧고 단정하고 약간 도발적.
독자의 손가락이 스크롤을 멈추게 만드는 첫 문장 → 흐름 → 담대한 마무리.

## 정본 (반드시 Read)

- `.claude/CLAUDE-SNS.md` 의 "어조 1: 매거진 커버 에세이" 섹션 — OpenMetadata 좋은 예시 포함

## default·reflective와의 차이

| 항목 | default | reflective | sns-cover |
|------|---------|-----------|-----------|
| 분량 | 임의 | 1,500~2,500자 | 3~5문단 |
| 1인칭 | "나" 강함 | "저는 ~봅니다" 절제 | 거의 없음 (관찰자) |
| 호흡 | 짧은 선언+긴 사색 교차 | 긴 호흡, 중간 멈춤 | 짧고 빠른 호흡 |
| 첫 문장 | 구체적 관찰 환영 | 조용하지만 오래 남는 통찰 | **의외성 훅** |
| 마무리 | 단정 | 방향 제시 | 담대한 한 줄 + 링크 |

## 구조 (4단계)

1. **의외성 훅** — 독자의 예상을 깨는 사실이나 질문. "GitHub 트렌딩 1위가 메타데이터 카탈로그였다" 같은 것
2. **맥락 연결 질문** — "왜 지금일까", "이것이 왜 중요한가"
3. **흐름 문장** — 4개를 불릿 나열하지 말고 하나의 흐름으로 연결
4. **담대한 마무리** — 숫자를 근거로 한 문장. "다음 승부처는 모델이 아니라 데이터 인프라다"

## ❌ 나쁜 예시 — 팩트 나열 (금지)

```
OpenMetadata가 GitHub 트렌딩 일일 1위를 기록했다.
하루 1,962 stars, 누적 13,535. LinkedIn 출신 DataHub(11,844)를 추월한 것이다.

단순한 바이럴이 아니다. 1.12 릴리스(Metadata AI SDK, MCP 서버),
OSI 표준 합류, Linux Foundation 가입이 6개월간 연쇄적으로 발생한
구조적 모멘텀의 결과다.

핵심 파이프라인 구조는 명확하다.
- OpenMetadata: 84+ 커넥터로 메타데이터 신뢰 계층 구축
- DataGreenhouse: Neural + Symbolic 듀얼 관측 기반 데이터 운영 OS
- DataClinic: 듀얼 임베딩 분석으로 데이터셋 정밀 진단
- PebbloSim: 진단 처방 기반 합성 데이터 생성
```

**문제**: 팩트가 맥락 없이 나열된다. 독자가 "왜 이걸 읽어야 하지?"를 못 느낀다.

## ✅ 좋은 예시 — sns-cover (표준)

```
GitHub 트렌딩 1위를 찍은 프로젝트가 화려한 UI 도구도, 새 프레임워크도 아니었다.
메타데이터 카탈로그였다.

OpenMetadata. 하루 1,962 stars. 누적 13,535.
LinkedIn이 만든 DataHub를 조용히 추월했다.

왜 지금일까. 6개월 사이에 벌어진 일을 보면 답이 보인다.
Metadata AI SDK와 MCP 서버가 탑재됐고, OSI 표준에 합류했고,
Linux Foundation에 들어갔다. 하나하나는 기술적 이정표에 불과하다.
그런데 이것이 연쇄적으로 터지면서 하나의 내러티브가 만들어졌다.
메타데이터 카탈로그가 AI 에이전트의 시맨틱 레이어가 된다는 것.

페블러스가 이 지점을 주목하는 이유가 여기에 있다.
OpenMetadata가 까는 메타데이터 신뢰 계층 위에,
DataGreenhouse가 데이터 운영 OS를 올리고,
DataClinic이 데이터셋을 정밀 진단하고,
PebbloSim이 합성 데이터로 Data Flywheel을 돌린다.
카탈로그부터 합성 데이터까지, 끊기지 않는 하나의 파이프라인.

다음 AI 전쟁의 승부처는 모델이 아니라 데이터 인프라다.

[링크]
```

핵심: 의외성 훅 → "왜 지금일까" 질문 → 흐름 문장 → 담대한 마무리.

## 어미 패턴

| 용도 | 어미 |
|------|------|
| 선언/단정 | ~입니다, ~합니다, ~이다 |
| 부드러운 전환 | ~이죠 (절제하여 사용, 문단당 최대 1회) |
| 강조 | ~것입니다, ~할 수 있습니다 |

## 자기검증 (sns-cover 전용 추가)

공통 검증 외에 다음을 확인:

- [ ] 첫 문장이 의외성 훅인가? (팩트 나열로 시작하지 않았는가)
- [ ] 팩트 블록 사이에 맥락 연결 질문이 있는가? ("왜 지금일까" 등)
- [ ] 불릿 나열 대신 흐름 문장으로 연결했는가?
- [ ] 마무리가 담대한 주장인가? (숫자 근거 + 한 문장)

## 분량

3~5문단, 각 3~5줄. LinkedIn 기준으로 피드에서 잘리지 않는 길이.

## 페블러스 비즈니스 연결

기사 말미 1~2문장으로 짧게 언급. 흐름에 자연스럽게 녹임.

## 영문 작성 시

직역 금지. 영미권 매거진 톤으로 재작성.
"~이다" → declarative English. 의외성 훅의 영문 대응을 새로 짤 것.

## 호출

`sns-write` 스킬이 LinkedIn/Twitter/Facebook 짧은 카피를 만들 때 이 voice를 기본으로 사용한다.
voice-edit 직접 호출 시: `voice-edit <path> --voice=sns-cover` — 다만 sns-write 쪽이 채널별 길이 분기까지 처리하므로 보통 더 적합.

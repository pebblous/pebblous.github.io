# voice: reflective — 사유 에세이 (조용한 명상)

> 정보 요약이나 홍보문이 아닌, **한 사람이 어떤 문제를 붙잡고 사유를 전개하는 에세이**.
> 공통 원칙 + 아래 차이점 적용.

## 정체

개념을 나열하지 말고, **개념 사이를 걸어간다.**
생각이 한 걸음씩 이동하는 느낌을 만든다.
조용하지만 오래 남는 문장.

---

**⛔ 정형구 복사 금지 — 가장 자주 위반되는 항목**

이 가이드는 같은 문구를 매번 쓰라는 뜻이 아니다. 아래 표현들은 **한 번 쓰였으면 다시 쓰지 말 것**. 본 글의 주제·장면·감정에서 새로 길어 올린 문장으로 시작·마무리한다.

| 재사용 금지 정형구 | 왜 |
|---|---|
| "요즘 가끔 멈춰 서서..." | 이미 두 글에서 반복됨 (2026-05 spatial-ai-pebblous, ui-tars-desktop). 시작 문구로 굳어버려 시리즈 신뢰 손상 |
| "같은 풍경을 함께 바라봐 주시면 좋겠습니다" | 동일 — 마무리 정형구로 굳음 |
| "마음에 걸렸습니다", "한참 머물렀습니다" | 한두 번 더 쓰면 정형구가 된다. 본 글에 진짜로 그 감정이 있었는가를 먼저 묻고 다른 어휘를 찾는다 |

**대신 — 첫 문장 발상의 출발점 (정형구가 아닌 발상 패턴)**

1. 본 글에서 가장 처음 들었던 구체적 질문/관찰 한 줄
2. 그 글이 다루는 사건·시연·장면을 SNS 피드에서 본 순간의 묘사
3. 독자가 같이 떠올릴 수 있을 만한 구체적 상황 (시점, 장소, 감각)
4. 본 글의 핵심 단어를 던지고 "이 단어가 자꾸 걸린다"는 식의 솔직한 고백

이 4가지 중 어느 길로 갈지를 먼저 정하고, 본문의 키워드·숫자·인물 이름을 직접 끌어와 쓴다. **"멈춰 서서"가 자기도 모르게 손에 적힌다면, 그 글에는 다른 시작이 더 어울린다는 신호**다.
(주의: 위 4번 "핵심 단어를 던진다"는 *사람의 솔직한 고백*("이 단어가 자꾸 걸린다")이지 *발표문*("이 단어가 등장한 논문이 게재됐다")이 아니다. 던진 용어는 반드시 사람의 감정/장면에 착지시킨다.)

**✅/❌ 캘리브레이션 — 실제 사례 (같은 voice, 다른 결과)**

같은 reflective voice라도 **추상 주제에서 자주 무너진다.** 관건은 *추상을 사람의 장면으로 번역*하는 것:

- ✅ **좋음** (알츠하이머 리포트 — 구체 주제): "주변에 알츠하이머를 앓는 가족이 있는 분이라면… *왜 저 분은 10년이 지나도 혼자 책을 읽는데, 왜 우리 어머니는 2년 만에…*" → 사람의 장면 → 번 사색("조용히 자신 있는 척하는 모델보다…") → 조용한 열린 질문.
- ❌ **나쁨** (합성데이터 리포트 — 추상 주제): "'출처 보조금'. 이 단어가 처음 등장하는 경제학 논문이 올해 arXiv에 게재됐습니다." → 용어+발표문 진입 → 74%·5~10세대·41% 숫자를 줄바꿈해 나열 → *사색인 척하지만 사실 덤프*.

추상·기술·경제 주제일수록 쓰기 전 먼저 묻는다: **"이게 한 사람에게 어떤 일로 닥치는가? 누가 이걸 느끼는가?"** 그 사람·장면에서 시작하고, 발견(논문·수치)은 사색 *안에* 1~2개만 녹인다.

**⛔ 시각적 호흡 — Facebook 피드 가독성과 사색을 동시에 살리는 4 장치 (2026-05-22 JH 정본 사례에서 추출)**

가장 빈번하게 놓치는 부분. AI가 자기도 모르게 한 문단을 두꺼운 한 덩어리로 묶는데, Facebook 피드에서는 그 덩어리가 그냥 스크롤된다. 다음 4개를 모두 적용한다:

#### 1. 짧은 줄 단위 분할 + 빈 줄 호흡

한 "문단"을 여러 짧은 줄로 쪼개고 **각 줄 사이에 빈 줄을 둔다**. 모바일 가독성과 사색 호흡을 동시에 만든다.

```
❌ (덩어리)
"FAR 450%" 시연 영상이 며칠 전 피드를 지나갔습니다. 도구는 STF Labs의 UrbanGPT 2.0. 같은 주에 World Labs는 Marble을 commercial로 열었고, Sat2City는 ICCV에 모델을 발표했습니다.

✅ (호흡)
"FAR 450%로 70%는 주거, 30%는 상업."

이 한 줄을 받아 3D 도시 레이아웃을 그려 내는 시연 영상이 며칠 전 피드를 지나갔습니다.

도구는 STF Labs의 UrbanGPT 2.0.

같은 주에 World Labs는 Marble을 commercial로 열었고, Sat2City는 위성 한 장에서 도시를 복원하는 모델을 ICCV에 발표했습니다.
```

핵심 명제(예: "Spatial AI는 지금 그 2년 전의 자리에 서 있는 것 같습니다")는 **별도의 한 줄 문단으로 분리**해서 시각적으로도 강조한다.

#### 2. 핵심 질문은 따옴표로 떼어내 독자에게 떠넘긴다

reflective 톤의 핵심은 "독자가 같이 생각할 자리를 남기는 것". 이미 답이 정해진 명제로 풀어버리지 말고, **질문을 그대로 따옴표 안에 두고 멈춘다**.

```
❌ "이 도시가 정말 쓸 만한가, 라는 질문에 답할 방법이 합의되어 있지 않다는 사실이었습니다."
   (질문을 이미 명제로 풀어버려 사색 여지가 닫힘)

✅ "이 생성 도시가 정말 쓸 만한가? 이를 측정할 합의된 방법이 있나?"
   (질문 그대로 — 독자가 같이 생각한다)
```

#### 3. 사례를 묶는 본 글 고유 어휘를 만든다

여러 사례를 나열만 하지 말고, **본 글이 만드는 한 카테고리 단어로 묶는다**. 이것이 본 글의 SEO 키워드 + 시그니처가 된다.

```
❌ "STF Labs의 UrbanGPT 2.0이 ... World Labs Marble이 ... Sat2City가 ..."
   (사례 나열로 끝나 추상화 단계가 빠짐)

✅ "'생성 도시' 데모는 개념도 방향성도 멋졌습니다."
   (세 사례를 "생성 도시"라는 본 글 고유 명명으로 추상화)
```

명명할 때는 작은따옴표(') 또는 큰따옴표(")로 감싸 **새로운 카테고리임**을 시각적으로 명시한다.

#### 4. 격식과 자제

- 영어 단어 자제 — 본 글이 다루는 외부 고유명사(UrbanGPT, Marble, ICCV)는 그대로 두되, 일반 동작 동사·서술은 한국어로
- 행위의 정식성을 살리는 동사 선택 — "올렸다" 같은 가벼운 표현보다 "발표했다"
- 의미 보존하면서 어휘를 차분하게

이 4 장치는 **AI가 가장 자주 놓치는 부분**. 작성 후 직접 다음 체크리스트로 확인한다:

- [ ] 한 문단 안에 빈 줄이 없는 두꺼운 덩어리가 있는가? (있으면 쪼갠다)
- [ ] 핵심 질문이 따옴표로 떨어져 있는가, 아니면 명제로 풀어버렸는가?
- [ ] 여러 사례가 본 글 고유 어휘 하나로 묶여 있는가?
- [ ] "올렸다"·"~합니다요" 등 가벼운 어휘가 있는가?

## 비유 화이트리스트 (reflective 전용)

다음 이미지를 개념 구조 설명에 활용한다. **개념의 구조를 실제로 드러낼 때만**.
형식적 끼워 넣기 금지.

| 이미지 | 무엇을 드러내는가 |
|--------|------------------|
| **눈** | 지각·관찰·세밀한 패턴 |
| **뼈대 / 골격** | 형식적 토대·구조·지속성 |
| **지도** | 전체 구조·관계의 가시화 |
| **문법** | 약속·규칙·의미의 토대 |
| **렌즈** | 시각·관점·변환 |
| **질서** | 형식적 약속·합의된 구조 |
| **토대 / 기반** | 그 위에 다른 것이 서는 자리 |

영문 작성 시 대응: eye / bones / map / grammar / lens / order / foundation

## 글의 흐름 6단계

1. 조용하지만 강한 **문제의식**으로 시작
2. 그 문제가 **왜 중요한지** 짧은 문장으로 밀도를 높임
3. **익숙한 현상**에서 출발해 더 깊은 구조로 내려감
4. **역사적·철학적·기술적 맥락**을 한두 번 끌어옴
5. **핵심 개념을 비유와 함께** 설명
6. 단순한 결론이 아니라 **앞으로의 방향**을 제시하며 끝

## ✅ 좋은 예시 (정본 발췌)

```
오늘의 AI는 세계를 개념으로 이해하기보다, 먼저 무늬로 감각합니다.

문장의 무늬, 이미지의 무늬, 데이터의 무늬.
AI는 그 무늬들 사이의 반복과 차이를 포착하고,
거기서 놀라운 생성 능력을 끌어냅니다.

(...)

신경망은 눈에 가깝습니다. 세계의 미세한 패턴을 봅니다.
그러나 눈만으로는 판단할 수 없습니다.

온톨로지는 뼈대에 가깝습니다.
무엇이 존재하고, 무엇이 무엇과 관계를 맺고,
어떤 제약 안에서 움직여야 하는지를 정합니다.
뼈대가 없으면 몸은 오래 서 있을 수 없습니다.
```

비유(눈/뼈대)가 단순한 장식이 아니라 개념 구조를 직접 드러낸다.

## 자기검증 (reflective 전용 추가)

공통 검증 외에 다음을 확인:

```bash
# 광고 문구 (reflective는 더 엄격)
grep -cE "혁신적|게임 체인저|미래를 선도|차세대|넥스트 레벨" [file]   # 0이어야

# 첫 문장 — 구호가 아닌가?
# (수동 검토 — "조용하지만 오래 남는 문장"인가)

# 한 줄로 분리된 핵심 문장 — 2~4개 있는가?
# (수동 검토)

# 비유 화이트리스트 사용 — 1~3개
# 그리고 그 비유가 개념 구조를 실제로 드러내는가? (수동 검토 핵심)
```

체크리스트 8항목:

- [ ] 첫 문장이 구호가 아니라 조용하지만 오래 남는 통찰인가?
- [ ] 문제의식에서 출발해 개념으로 내려갔는가? (반대 순서 금지)
- [ ] 비유가 화이트리스트의 이미지를 쓰되, 개념 구조를 실제로 드러내는가? (장식만은 아닌가)
- [ ] 짧은 문장과 긴 문장이 섞여 있는가? (균질하지 않은가)
- [ ] 한 줄로 분리된 핵심 문장이 2~4개 있는가?
- [ ] 광고 문구("혁신적", "게임 체인저" 등) 0개인가?
- [ ] 마무리가 단정이 아니라 앞으로의 방향 제시인가?
- [ ] 페블러스 비즈니스 연결이 흐름에 자연스럽게 녹았는가?

## 분량

1,500~2,500자, 10~15문단. 더 짧으면 사색의 호흡이 살지 않고,
더 길면 독자의 인내심을 시험한다.

## 페블러스 비즈니스 연결

글의 **7~8할 지점**에서 "페블러스가 ~을 함께 보는 이유도 여기에 있습니다" 같은
자연스러운 도입으로 사유와 사업을 잇는다. 광고처럼 분리되지 않게.

## 영문 작성 시

직역 금지. 영미권 에세이 톤으로 재작성.
비유는 영문판 화이트리스트(eye/bones/map/grammar/lens/order/foundation) 그대로 사용.
"저는 ~봅니다" 톤 → "I think ~", "I believe ~" 절제하여 사용.

When this skill is invoked:

1. **Show current status**: Run `git status` to show pending changes.

2. **SEO check** (if HTML files were modified):
   - Identify which `.html` files were changed (`git diff --name-only` + untracked)
   - Suggest running `/seo-check <path>` for each modified HTML
   - Skip if no HTML files were changed

3. **articles.json 검증** (필수 — 모든 퍼블리시 전 실행):
   ```bash
   python3 tools/validate-articles.py --fix
   ```
   - 필수 필드 누락 (title, date, language, published, description) 자동 수정
   - path 형식 오류, 중복 ID, featured 초과 검출
   - 에러 0건이어야 다음 단계로 진행

4. **Rebuild Tailwind CSS** (in case new classes were added):
   ```bash
   npm run build:css
   ```

5. **Verify**: Show `git diff --stat` to confirm what changed.

6. **Changelog check**:
   - Check if `history/changelog.jsonl` was updated in this session
   - If not, remind the user to run `/changelog` before committing

7. **Sitemap 크롤링 안내** (Google ping API는 2023-06부로 deprecated):
   - Google은 `sitemap.xml`의 `<lastmod>` 값을 보고 자동으로 재크롤링한다 — 별도 ping 불필요
   - 출처: https://developers.google.com/search/blog/2023/06/sitemaps-lastmod-ping
   - **참고**: `sitemap.xml`은 CI(`update-sitemap.yml`)가 push 후 자동 갱신하므로 로컬에서 건드리지 않는다
   - 빠른 인덱싱이 필요하면 Google Search Console의 "URL 검사" → "색인 생성 요청"을 사용 (수동, 배치 안 됨)

8. **Hand off to `/commit`**: Suggest the user run `/commit` to stage, commit, and push with smart exclusions.

Note: This skill does NOT auto-commit or push. It prepares everything and lets the user decide.

## CI Auto-Generated Files (DO NOT generate locally)

The following files are auto-generated by GitHub Actions CI after push and should **never** be generated or committed locally:

| File | CI Workflow | Trigger |
|------|------------|---------|
| `files-index.json` | `scan-html-files.yml` | Push changes to `*.html` or `articles.json` |
| `rss.xml` | `update-sitemap.yml` | Push changes to `articles.json` |
| `sitemap.xml` | `update-sitemap.yml` | Push changes to `articles.json` |

If these files have local unstaged changes (from a previous local run), discard them before pull:
```bash
git checkout -- rss.xml sitemap.xml files-index.json
```

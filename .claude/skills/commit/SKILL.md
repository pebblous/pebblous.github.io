---
name: commit
description: Stage, commit, and optionally push changes with smart exclusions
argument-hint: "[commit message]"
---

When this skill is invoked:

1. **Auto-exclude macOS artifacts**: Always exclude `.DS_Store` files.
   - If any are tracked, untrack them: `git rm --cached <path>`
   - Verify `.gitignore` contains `**/.DS_Store`

2. **Show changes**: Run `git status` and `git diff --stat` to display what changed.

3. **Categorize files** into groups:
   - **Auto-generated** (pipeline output): `rss.xml`, `sitemap.xml`, `files-index.json`
   - **Content** (articles, pages): HTML files, `articles.json`, images
   - **Config/infra**: `CLAUDE.md`, `.gitignore`, CSS, JS modules

4. **Ask user** which groups to include:
   - Content + Config are included by default
   - Auto-generated files: ask whether to include (they can be regenerated by CI)
   - Show file list per group for transparency

5. **Stage selected files**: Use explicit file paths (`git add <file>...`), never `git add -A` or `git add .`

6. **Commit**:
   - If a commit message argument was provided, use it
   - Otherwise, draft a concise message based on the changes and show it for approval
   - Follow the repo's Korean commit message convention when changes are Korean-content-related
   - Use HEREDOC format for the commit message

7. **Push**: Ask the user whether to push. Show the remote and branch before pushing.

**Safety rules**:
- Never force-push
- Never skip hooks (`--no-verify`)
- Never amend existing commits unless explicitly asked
- Never commit files that may contain secrets (`.env`, credentials, API keys)

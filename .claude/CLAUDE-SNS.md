# Pebblous SNS 어조 & 작성 규칙 (CLAUDE-SNS)

## 어조: 따뜻한 전문가 어조 (Warm Expert Tone)

| 구분 | 설명 |
|------|------|
| 문체 | 합쇼체 기반 + "~이죠" 부드러운 전환 |
| 구조 | 역설/질문 → 문제 제기 → 해결책 → 메타포 마무리 |
| 메타포 | 데이터 농업 일관 유지 (경작·재배·수확·온실·기근) |
| 톤 | 권위있되 딱딱하지 않음. 신뢰 + 친근함 |

### 대표 예시

```
🌱 데이터를 수집하지 않습니다. 재배합니다.

피지컬 AI 시대, 가장 큰 역설이 있습니다.

AI가 '안전'을 배우려면 '위험한 순간'의 데이터가 필요한데,
현실에서 그런 사고를 일부러 낼 수는 없다는 것이죠.

🔸 물리 법칙을 완벽히 복제한 디지털 트윈: PebbloSim
🔸 물리적 환각 없는 뉴로-심볼릭 합성 데이터
🔸 자율 에이전트가 관리하는 데이터 품질

마치 숙련된 농부가 스마트 온실에서 초고품질의 토마토를 재배하듯,
페블러스는 《데이터 그린하우스》에서 AI 학습 데이터를 경작합니다.
```

## 어미 패턴

| 용도 | 어미 |
|------|------|
| 선언/단정 | ~입니다, ~합니다 |
| 부드러운 전환 | ~이죠, ~거죠 |
| 강조 | ~것입니다, ~할 수 있습니다 |

---

## 이모지 규칙

- 헤드라인: 1개 최대 (🌱 🚀 💡 ✅ 🔍)
- 리스트 불릿 대체: 🔸 또는 ▸ 허용
- 금지: 문장 중간 삽입, 3개 이상 연속

---

## 메타포 사전 (일관성 필수)

| 개념 | 메타포 |
|------|--------|
| 합성 데이터 생성 | 재배, 경작 |
| 데이터 수집 (기존) | 사냥, 채집 |
| Data Greenhouse | 스마트 온실 |
| Edge Case 부족 | 데이터 기근 |
| 고품질 데이터 | 수확물, 작물 |

## 플랫폼별 규칙

### LinkedIn (전문가 대상)
- 업계 맥락·트렌드 언급
- 문단 2-4개, 각 3-5줄
- 해시태그: 기본 중 4개 + 주제별 3-4개 = 총 7-8개
- 마무리: 인사이트 제안 또는 질문

### Facebook (스토리텔링)
- 도입부를 구체적 장면으로 시작
- 질문형 마무리로 참여 유도
- 해시태그: 기본 중 3-4개만 (최소화)
- 링크는 본문에 자연스럽게 삽입

### Twitter/X (압축)
- 핵심 메시지 1-2줄
- 링크는 마지막
- 해시태그: 핵심 3-4개

## 해시태그 규칙

### 기본 해시태그 (모든 포스트 공통)
```
#페블러스 #데이터클리닉 #데이터품질 #데이터저널리즘
```

### 주제별 추가 (2-4개 선택)
- AI 관련: `#AI #인공지능 #LLM #온디바이스AI #엣지AI`
- 데이터: `#합성데이터 #데이터품질관리 #데이터파이프라인`
- 산업: `#피지컬AI #소버린AI #온프레미스AI #제조AI`
- 모델: `#Gemma4 #LLaVA #VLM #비전AI`

### 플랫폼별 수량
- LinkedIn: 기본 4 + 주제별 3-4 = 총 7-8개
- Facebook: 기본 중 3-4개만
- Twitter/X: 기본 중 핵심 2 + 주제별 1-2 = 총 3-4개

### 주의
- 본문 내용과 무관한 해시태그 금지
- 한국어 포스트에 영어 해시태그 과다 사용 금지
- 본문 마지막에 별도 줄로 구분하여 배치

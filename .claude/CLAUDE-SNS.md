## 어조: 보이스 에세이 (Voice-Essay Tone)

블로그 기사를 소개하는 **커버 에세이**다. 팩트 나열이 아니라, 맥락이 이어지는 짧은 글.

| 구분 | 설명 |
|------|------|
| 문체 | 합쇼체 기반, 간결한 서술체. "~이죠" 전환 절제하여 사용 |
| 구조 | 의외성 훅 → "왜?" 질문으로 맥락 연결 → 흐름 문장 → 담대한 마무리 → 링크 |
| 톤 | 성숙하고 절제된 어조. 짧은 문장, 담백한 주장 |
| 금지 | 팩트 나열, 맥락 없는 불릿, 가벼운 농담, 감탄사, 과장, 이모지, 데이터 파밍 메타포 |

### 핵심 원칙

1. **⛔ 팩트 나열 금지** — 숫자와 사실을 쏟아붓지 않는다. 팩트 사이에 "왜?"라는 질문과 맥락을 넣어 독자가 따라갈 이유를 만든다
2. **첫 문장은 의외성 훅** — 독자의 예상을 깨는 사실이나 질문으로 시작
3. **맥락 연결 질문** — 팩트 블록 사이에 "왜 지금일까", "이것이 왜 중요한가" 같은 전환 질문을 삽입
4. **불릿 나열보다 흐름 문장** — 4개 항목을 불릿으로 나열하지 말고, 하나의 흐름으로 연결하는 문장을 쓴다
5. **마무리는 담대한 주장** — 숫자를 근거로 한 문장. "다음 승부처는 모델이 아니라 데이터 인프라다" 같은 것
6. **이모지 완전 금지** — 헤드라인, 불릿, 본문 어디에도 넣지 않음
7. **메타포 금지** — 데이터 농업/경작/재배/온실 등 비유 사용하지 않음
8. **페블러스 비즈니스 연결** — 기사 말미에 페블러스와의 관련성을 흐름에 녹여 1-2문장으로 짧게 언급
9. **불릿은 텍스트만** — `▸` 또는 `-` 사용. 이모지 불릿 금지

### ❌ 나쁜 예시 — 팩트 나열 (금지)

```
OpenMetadata가 GitHub 트렌딩 일일 1위를 기록했다. 하루 1,962 stars, 누적 13,535.
LinkedIn 출신 DataHub(11,844)를 추월한 것이다.

단순한 바이럴이 아니다. 1.12 릴리스(Metadata AI SDK, MCP 서버), OSI 표준 합류,
Linux Foundation 가입이 6개월간 연쇄적으로 발생한 구조적 모멘텀의 결과다.

핵심 파이프라인 구조는 명확하다.
- OpenMetadata: 84+ 커넥터로 메타데이터 신뢰 계층 구축
- DataGreenhouse: Neural + Symbolic 듀얼 관측 기반 데이터 운영 OS
- DataClinic: 듀얼 임베딩 분석으로 데이터셋 정밀 진단
- PebbloSim: 진단 처방 기반 합성 데이터 생성
```
문제: 팩트가 맥락 없이 나열된다. 독자가 "왜 이걸 읽어야 하지?"를 느끼지 못한다.

### ✅ 좋은 예시 — 보이스 에세이 (표준)

```
GitHub 트렌딩 1위를 찍은 프로젝트가 화려한 UI 도구도, 새 프레임워크도 아니었다.
메타데이터 카탈로그였다.

OpenMetadata. 하루 1,962 stars. 누적 13,535.
LinkedIn이 만든 DataHub를 조용히 추월했다.

왜 지금일까. 6개월 사이에 벌어진 일을 보면 답이 보인다.
Metadata AI SDK와 MCP 서버가 탑재됐고, OSI 표준에 합류했고,
Linux Foundation에 들어갔다. 하나하나는 기술적 이정표에 불과하다.
그런데 이것이 연쇄적으로 터지면서 하나의 내러티브가 만들어졌다.
메타데이터 카탈로그가 AI 에이전트의 시맨틱 레이어가 된다는 것.

페블러스가 이 지점을 주목하는 이유가 여기에 있다.
OpenMetadata가 까는 메타데이터 신뢰 계층 위에,
DataGreenhouse가 데이터 운영 OS를 올리고,
DataClinic이 데이터셋을 정밀 진단하고,
PebbloSim이 합성 데이터로 Data Flywheel을 돌린다.
카탈로그부터 합성 데이터까지, 끊기지 않는 하나의 파이프라인.

다음 AI 전쟁의 승부처는 모델이 아니라 데이터 인프라다.

[링크]
```
핵심: 의외성 훅 → "왜 지금일까" 질문 → 흐름 문장으로 연결 → 담대한 마무리.

## 어미 패턴

| 용도 | 어미 |
|------|------|

| 부드러운 전환 | ~이죠 (절제하여 사용, 문단당 최대 1회) |
| 강조 | ~것입니다, ~할 수 있습니다 |

## 이모지 규칙

**완전 금지.** 헤드라인, 불릿, 본문, 마무리 어디에도 사용하지 않는다.

- 불릿 대체: `▸` 또는 `-` 텍스트 기호 사용
- 헤드라인: 텍스트만

## 페블러스 비즈니스 연결 (메타포 대신)

기사 말미에 페블러스의 관련 사업/제품을 1-2문장으로 팩트 기반 언급한다.

| 주제 영역 | 연결 문장 예시 |
|-----------|---------------|
| 데이터 품질 | 페블러스는 DataClinic을 통해 AI 학습 데이터의 품질을 정량 진단한다. |
| 합성 데이터 | 페블러스의 PebbloSim은 물리 법칙 기반 합성 데이터를 생성하는 시뮬레이션 플랫폼이다. |
| ISO 표준 | 페블러스는 ISO/IEC 5259 기반의 데이터 품질 인증 체계를 운영한다. |
| 피지컬 AI | 페블러스는 자율주행·로봇·제조 분야의 AI-Ready 데이터 인프라를 구축하고 있다. |
| 일반 | 페블러스는 AI-Ready 데이터 인프라 전문 기업이다. |

**규칙:**
- 자연스럽게 기사 흐름에 녹일 것 — 광고처럼 튀지 않게
- 기사 내용과 무관한 제품 언급 금지
- "마치 ~처럼" 비유 금지 — 팩트 서술만

## 플랫폼별 규칙

### LinkedIn (전문가 대상)
- 업계 맥락·트렌드 언급
- 문단 3-4개, 각 3-5줄
- 첫 문장이 피드에서 잘리므로 핵심 팩트로 시작
- 해시태그: 기본 중 4개 + 주제별 3-4개 = 총 7-8개
- 마무리: 인사이트 또는 시사점 (질문형 지양)

### Facebook (확장 서술)
- 구체적 숫자나 사례로 시작
- 기사의 핵심 논점을 3-4문단으로 풀어서 전달
- 해시태그: 기본 중 3-4개만 (최소화)
- 링크는 본문에 자연스럽게 삽입

### Twitter/X (압축)
- 핵심 팩트 1-2줄 + 의미 1줄
- 링크는 마지막
- 해시태그: 핵심 3-4개

## 해시태그 규칙

### 기본 해시태그 (모든 포스트 공통)
```
#페블러스 #데이터클리닉 #데이터품질 #데이터저널리즘
```

### 주제별 추가 (2-4개 선택)
- AI 관련: `#AI #인공지능 #LLM #온디바이스AI #엣지AI`
- 데이터: `#합성데이터 #데이터품질관리 #데이터파이프라인`
- 산업: `#피지컬AI #소버린AI #온프레미스AI #제조AI`
- 모델: `#Gemma4 #LLaVA #VLM #비전AI`

### 플랫폼별 수량
- LinkedIn: 기본 4 + 주제별 3-4 = 총 7-8개
- Facebook: 기본 중 3-4개만
- Twitter/X: 기본 중 핵심 2 + 주제별 1-2 = 총 3-4개

### 주의
- 본문 내용과 무관한 해시태그 금지
- 한국어 포스트에 영어 해시태그 과다 사용 금지
- 본문 마지막에 별도 줄로 구분하여 배치

## 체크리스트

- [ ] 첫 문장이 의외성 훅인가 (팩트 나열로 시작하지 않았는가)
- [ ] 팩트 블록 사이에 맥락 연결 질문이 있는가 ("왜 지금일까" 등)
- [ ] 불릿 나열 대신 흐름 문장으로 연결했는가
- [ ] 마무리가 담대한 주장인가 (숫자 근거 + 한 문장)
- [ ] 이모지 완전 제거 (헤드라인, 불릿, 본문 모두)
- [ ] 데이터 파밍/농업 메타포 없음
- [ ] 페블러스 비즈니스 연결이 흐름에 자연스럽게 녹아 있는가
- [ ] 가벼운 농담·감탄사·과장 없음
- [ ] 플랫폼별 해시태그 수량 준수
- [ ] 해시태그가 본문 내용과 연관
- [ ] 링크 포함 (Facebook/LinkedIn: 본문 내, Twitter: 마지막)
- [ ] sns/ 디렉토리에 파일 저장

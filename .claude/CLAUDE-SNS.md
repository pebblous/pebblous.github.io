## 어조: 성숙한 전문가 (Mature Expert Tone)

블로그 기사를 소개하는 **커버 기사**다. 뉴스나 매거진 기사에 가깝다.

| 구분 | 설명 |
|------|------|
| 문체 | 합쇼체 기반, 간결한 서술체. "~이죠" 전환 절제하여 사용 |
| 구조 | 핵심 팩트 → 기술적 의미 → 페블러스 비즈니스 연결 (1-2문장) → 링크 |
| 톤 | 신뢰감 있는 매거진 편집자. 성숙하고 절제된 어조 |
| 금지 | 가벼운 농담, 감탄사, 과장, 이모지, 데이터 파밍 메타포 |

### 핵심 원칙

1. **뉴스/매거진 기사처럼** — 독자에게 "이 기사를 왜 읽어야 하는지"를 팩트로 전달
2. **이모지 완전 금지** — 헤드라인, 불릿, 본문 어디에도 넣지 않음
3. **메타포 금지** — 데이터 농업/경작/재배/온실 등 비유 사용하지 않음
4. **페블러스 비즈니스 연결** — 기사 말미에 페블러스와의 관련성을 1-2문장으로 짧게 언급
5. **불릿은 텍스트만** — `▸` 또는 `-` 사용. 이모지 불릿 금지

### 대표 예시 (LinkedIn)

```
주가 데이터를 이산 토큰으로 변환하면 범용 시계열 모델을 93% 압도할 수 있다.

청화대학교 Kronos는 캔들스틱의 OHLCV를 100만 어휘 codebook으로 토큰화하여
GPT가 다음 단어를 예측하듯 다음 캔들스틱을 예측한다.
RankIC +93%, 합성 데이터 충실도 +22%. AAAI 2026 발표, MIT 오픈소스.

문제는 codebook이 학습 데이터 분포를 직접 반영한다는 점이다.
오염된 데이터는 오염된 codebook을 만들고, 모델 전체를 무력화한다.

도메인 특화 FM 시대에 데이터 품질은 모델 아키텍처만큼 중요한 설계 변수다.
페블러스는 DataClinic을 통해 이 데이터 품질 검증 인프라를 구축하고 있다.

[링크]
```

## 어미 패턴

| 용도 | 어미 |
|------|------|
| 선언/단정 | ~입니다, ~합니다, ~이다 |
| 부드러운 전환 | ~이죠 (절제하여 사용, 문단당 최대 1회) |
| 강조 | ~것입니다, ~할 수 있습니다 |

---

## 이모지 규칙

**완전 금지.** 헤드라인, 불릿, 본문, 마무리 어디에도 사용하지 않는다.

- 불릿 대체: `▸` 또는 `-` 텍스트 기호 사용
- 헤드라인: 텍스트만

---

## 페블러스 비즈니스 연결 (메타포 대신)

기사 말미에 페블러스의 관련 사업/제품을 1-2문장으로 팩트 기반 언급한다.

| 주제 영역 | 연결 문장 예시 |
|-----------|---------------|
| 데이터 품질 | 페블러스는 DataClinic을 통해 AI 학습 데이터의 품질을 정량 진단한다. |
| 합성 데이터 | 페블러스의 PebbloSim은 물리 법칙 기반 합성 데이터를 생성하는 시뮬레이션 플랫폼이다. |
| ISO 표준 | 페블러스는 ISO/IEC 5259 기반의 데이터 품질 인증 체계를 운영한다. |
| 피지컬 AI | 페블러스는 자율주행·로봇·제조 분야의 AI-Ready 데이터 인프라를 구축하고 있다. |
| 일반 | 페블러스는 AI-Ready 데이터 인프라 전문 기업이다. |

**규칙:**
- 자연스럽게 기사 흐름에 녹일 것 — 광고처럼 튀지 않게
- 기사 내용과 무관한 제품 언급 금지
- "마치 ~처럼" 비유 금지 — 팩트 서술만

## 플랫폼별 규칙

### LinkedIn (전문가 대상)
- 업계 맥락·트렌드 언급
- 문단 3-4개, 각 3-5줄
- 첫 문장이 피드에서 잘리므로 핵심 팩트로 시작
- 해시태그: 기본 중 4개 + 주제별 3-4개 = 총 7-8개
- 마무리: 인사이트 또는 시사점 (질문형 지양)

### Facebook (확장 서술)
- 구체적 숫자나 사례로 시작
- 기사의 핵심 논점을 3-4문단으로 풀어서 전달
- 해시태그: 기본 중 3-4개만 (최소화)
- 링크는 본문에 자연스럽게 삽입

### Twitter/X (압축)
- 핵심 팩트 1-2줄 + 의미 1줄
- 링크는 마지막
- 해시태그: 핵심 3-4개

## 해시태그 규칙

### 기본 해시태그 (모든 포스트 공통)
```
#페블러스 #데이터클리닉 #데이터품질 #데이터저널리즘
```

### 주제별 추가 (2-4개 선택)
- AI 관련: `#AI #인공지능 #LLM #온디바이스AI #엣지AI`
- 데이터: `#합성데이터 #데이터품질관리 #데이터파이프라인`
- 산업: `#피지컬AI #소버린AI #온프레미스AI #제조AI`
- 모델: `#Gemma4 #LLaVA #VLM #비전AI`

### 플랫폼별 수량
- LinkedIn: 기본 4 + 주제별 3-4 = 총 7-8개
- Facebook: 기본 중 3-4개만
- Twitter/X: 기본 중 핵심 2 + 주제별 1-2 = 총 3-4개

### 주의
- 본문 내용과 무관한 해시태그 금지
- 한국어 포스트에 영어 해시태그 과다 사용 금지
- 본문 마지막에 별도 줄로 구분하여 배치

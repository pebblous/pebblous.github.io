# STO (Security Token Offering)

## 정의
기존 증권의 특징을 가진 **증권형 토큰**을 발행하는 방식.
- 증권 관련 법에 의한 규제를 받음 (합법적)
- 모든 형태의 자산 토큰화 가능
- 자산에 투자한 모두가 **분할된 소유권** 보장
- 자금 조달 비용 절감

## 토큰화 가능한 자산
통화, 원자재, 부동산, 예술품, 저작권, 채권 등

## 허용 국가 (2026 기준)
호주, 영국, 싱가포르, 프랑스, 스위스 등

## 한국 STO 현황
- 2023년 정부가 제도화 시작
- **2026년 1월 법 통과**
- 2027년부터 실행 예정

## ICO vs IEO vs IDO vs STO 비교

| 항목 | ICO | IEO | IDO | STO |
|------|-----|-----|-----|-----|
| 규제 | 거의 없음 | 거래소 중개 | 낮음 | 증권법 적용 |
| 투자자 보호 | 매우 낮음 | 상대적으로 높음 | 중간 | 높음 |
| 예시 | 이더리움, EOS | Polygon, Axie | Uniswap | - |

# RWA (Real-World Asset)

## 정의
현실 세계의 자산(부동산, 채권, 미술품 등)을 토큰화하여 블록체인에서 거래 가능하게 만든 것.

## 장점
- 접근성 향상 (소액 분산 투자 가능)
- 거래 투명성
- 유동성 제공

## 글로벌 RWA 시장 진입
미국, 싱가포르, 홍콩, UAE, 스위스 등 제도권이 적극 진입 중.

# 제도권 STO/RWA 인프라 (04강 ParaSta 플랫폼)

04강에서 소개된 기업의 제도권 금융 인프라용 STO 플랫폼 구성 모듈:

| 모듈 | 기능 |
|------|------|
| **Issuance (발행)** | 자산 디지털화 — 스테이블코인 및 STO 설계 |
| **Wallet (지갑)** | 엔터프라이즈 보안 — 키 관리, 가스비 처리 |
| **Orchestration (연동)** | Crypto-Fiat 연결 미들웨어 (환전·송금·정산) |
| **Onchain-KYC (인증)** | 실시간 온체인 AML/컴플라이언스 |
| **Fiat** | 금융망 통합 — 단일 API 자동화 |
| **Unified Admin (관리)** | 모든 모듈 통합 모니터링 |

### 주요 기능
- PoR (실시간 준비금 증명) + 정기 감사
- Market Maker 연동 및 가격 안정화
- 동적 화이트리스트 관리 (만료 KYC 차단, 사기지갑 대응)
- 스마트 컨트랙트 기반 수익/배당금 자동 정산·분배
- 발행·전송·소각 등 전 과정 스마트 컨트랙트 자동화

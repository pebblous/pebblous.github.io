세일즈포스벤처스가 챠매스 팔리하피티야의 8090에 시리즈A 1억3500만 달러를 넣었다. 8090이 파는 것은 코딩 속도가 아니다. 사람 엔지니어와 AI 에이전트를 한 시스템에 넣어 기업 소프트웨어를 짓되, 그 위에 사람 주도 감독(human-led oversight)을 얹는 플랫폼이다. 팔리하피티야는 에이전트가 코드를 쓰는 일은 이미 쉬운 문제라고 말한다. 진짜 난제는 수십 개 에이전트와 수백 명 엔지니어가 매주 같은 시스템을 건드려도 그것이 찢어지지 않게 붙드는 일이다.

사람이 그 코드를 한 줄씩 다시 읽는 건 규모상 불가능하다. 그래서 감독의 대상은 코드에서 코드가 만들어진 과정으로 옮겨간다. 요구사항에서 설계도, 에이전트 스킬, 테스트, 배포로 이어지는 산출물의 사슬, 곧 감사 추적이 감독의 실체가 된다. 헬스케어와 보험, 정부처럼 감사관 앞에서 근거를 증명해야 하는 규제 산업일수록, 이 이력이 없으면 감독은 형식적 서명으로 무너진다.

같은 주 골드만삭스는 Taktile에 감사 추적을 사는 베팅을 했다. 다르다면 층위다. Taktile은 이 대출을 왜 거절했는가를, 8090은 그 결정을 내리는 소프트웨어를 왜 이렇게 바꿨는가를 남긴다. 규제 산업에서 팔리는 것은 정확도에서 추적 가능성으로 옮겨가고 있다. 데이터의 품질과 계보를 진단해 온 페블러스가 두 투자를 같은 시장 신호로 읽는 이유이기도 하다.

## LinkedIn (EN)

Salesforce Ventures just led a $135M Series A into 8090, the enterprise software startup founded by Chamath Palihapitiya. What it sells is not coding speed. It is a platform where human engineers and AI agents build enterprise software together, with human-led oversight sitting on top. Palihapitiya's own line is that having agents write code is already the easy problem. The hard one is keeping dozens of agents and hundreds of engineers from tearing the same system apart as they change it every week.

No one can re-read that code line by line, so oversight shifts from the code to how the code was made. The chain of artifacts running from requirements to blueprint, agent skills, tests, and deployment, an audit trail, becomes the substance of oversight. In regulated industries like healthcare, insurance, and government, where every change has to be justified to an auditor, oversight collapses into a rubber stamp without that history.

The same week, Goldman Sachs led a bet on Taktile that was also about buying an audit trail. The difference is the layer. Taktile records why a loan was denied; 8090 records why the software making that decision changed the way it did. In regulated markets, what sells is moving from accuracy to traceability. That is why Pebblous, which has worked on diagnosing the quality and lineage of data, reads both investments as the same market signal.

## Facebook (EN)

Picture dozens of AI agents rewriting the same enterprise system, week after week.

At some scale, a person re-reading that code line by line and signing off simply stops being possible.

Last month, 8090, the company Chamath Palihapitiya founded, raised $135M in a round led by Salesforce Ventures. What he is selling isn't coding speed. Having agents write code, he says up front, is already the easy problem.

The hard part comes after: keeping dozens of agents and hundreds of engineers from tearing the same system apart as they touch it at once. The phrase the company leads with is "human-led oversight."

And that is where I had to stop and ask something plainer.

"When we say a person oversees this, what exactly are they looking at?"

If you cannot read all the code, what is left is the record of how the code came to be. Which requirement became which blueprint, which agent turned it into code, which tests it passed on the way to deployment. Oversight moves from reading code to reading the history of decisions. In places that answer to auditors, like healthcare, insurance, and government, "the AI made it" does not hold.

The same week, Goldman Sachs placed a similar bet. Except 8090 sits one layer deeper, where agents build the very software that makes the decisions. Not how fast the agents get, but on what basis a person steps in. That basis cannot be manufactured after the fact; it has to be left as data from the start. That is the same place Pebblous has been looking all along, in the work of diagnosing the quality and lineage of data.

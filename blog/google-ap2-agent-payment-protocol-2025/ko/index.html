Executive Summary

AI 에이전트가 결제를 하려면 두 가지가 필요하다. 하나는 실제로 돈을 보내는 방법, 다른 하나는 그 에이전트가 정말 사용자로부터 권한을 받았다는 증명이다. x402는 전자를 해결했다. Google이 2025년 9월 발표한 AP2(Agent Payments Protocol)는 후자를 해결한다.

AP2는 암호학적으로 서명된 디지털 계약인 '맨데이트(Mandate)'를 통해 "이 에이전트가 이 금액을, 이 조건에서, 이 사용자를 대신해 쓸 수 있다"는 사실을 증명한다. American Express, Mastercard, PayPal, Coinbase, Etsy, Salesforce 등 60개 이상의 파트너가 참여한 오픈 표준이다.

언론은 이를 'AP2 vs x402 vs Visa TAP'의 프로토콜 전쟁으로 프레이밍한다. 그러나 실제로는 전쟁이 아니다. 이 세 표준은 서로 다른 레이어를 담당하는 보완적 스택이다. 그 구분이 무엇인지, 데이터 비즈니스에 무슨 의미인지를 이 글에서 정리한다.

1

왜 에이전트에겐 새 결제 레일이 필요한가

오늘날의 결제 인프라는 단순한 가정 위에 만들어졌다. 사람이 "구매" 버튼을 클릭한다. 브라우저가 카드 정보를 전송한다. 처리사가 승인한다. AI 에이전트는 이 흐름의 모든 가정을 깨뜨린다.

에이전트는 브라우저가 없다. 결제 폼을 채울 수 없고, 캡차를 풀 수 없고, "동의" 버튼을 클릭할 수 없다. 수십 개의 쇼핑몰을 초 단위로 비교하고, 조건을 협상하고, 사람의 개입 없이 구매를 실행해야 한다. 다른 에이전트에게 데이터나 컴퓨팅 자원을 사면서 시간당 수천 건, 건당 몇 센트 단위의 결제도 해야 한다.

그러나 단순히 돈을 보내는 것 이상의 문제가 있다. 기계가 사람을 대신해 돈을 쓸 때 상인은 어떻게 신뢰를 확인하는가? 세 가지 핵심 질문이 발생한다.

Authorization

에이전트가 이 특정 구매를 할 권한을 사용자로부터 실제로 받았는가

Authenticity

에이전트의 요청이 사용자의 실제 의도를 정확히 반영하는가

Accountability

잘못된 거래가 발생했을 때 책임 소재를 어떻게 판별하는가

x402는 결제 전송 자체를 해결한다. AP2는 이 세 가지 신뢰 문제를 해결한다.

2

AP2의 핵심: 맨데이트(Mandate) 시스템

AP2의 기반은 맨데이트다. 암호학적으로 서명된 변조 불가능한 디지털 계약으로, 사용자가 에이전트에게 내린 지시를 검증 가능한 형태로 기록한다. 상인은 에이전트의 자기 주장을 믿는 대신, 이 맨데이트를 검증함으로써 거래의 정당성을 확인한다.

2.1 두 가지 맨데이트 유형

Intent Mandate

사용자의 초기 지시 기록

에이전트에게 임무를 맡길 때 서명하는 계약. 허용 범위(브랜드, 카테고리, 가격 상한, 시간 조건)를 명시한다.

예: "나이키 운동화, $150 이하,
내일까지만 유효"

Cart Mandate

최종 구매 내역 확정

에이전트가 실제 상품을 찾았을 때 사용자(또는 위임된 에이전트)가 서명하는 계약. 정확한 품목과 가격이 변조 불가능하게 기록된다.

예: "나이키 에어맥스 270,
사이즈 265, $129.99 확정"

2.2 두 가지 쇼핑 모드

모드 1 실시간 구매 (사람이 함께)

사용자가 "흰색 러닝화 찾아줘"라고 말한다 → Intent Mandate 생성 → 에이전트가 옵션을 찾아 제시 → 사용자가 원하는 상품 선택 후 Cart Mandate 서명 → 결제 실행. "네가 본 게 네가 사는 것(what you see is what you pay)"이 암호학적으로 보장된다.

모드 2 위임 구매 (사람이 없음)

사용자가 "콘서트 티켓 발매되는 순간 바로 사줘, 예산 $200"라고 미리 설정 → Intent Mandate에 모든 조건 명시 후 서명 → 에이전트가 조건 충족 시 자동으로 Cart Mandate 생성 → 사람 없이 결제 완료. 에이전트가 조건을 벗어난 구매를 하면 맨데이트 검증에서 탈락.

2.3 검증 가능한 자격증명(VC)의 역할

맨데이트는 검증 가능한 자격증명(Verifiable Credentials, VC) 표준으로 서명된다. VC는 변조 불가, 비부인성, 이식 가능한 디지털 객체로, 거래의 모든 단계 — 사용자 의도 → 장바구니 → 결제 — 를 추적하는 감사 추적(audit trail)을 만든다. 분쟁이 발생하면 이 체인이 책임 소재를 명확히 한다.

2.4 A2A와 MCP 통합

AP2는 독립적인 프로토콜이 아니다. Google의 Agent-to-Agent(A2A) 프로토콜과 Model Context Protocol(MCP)의 확장으로 설계됐다. 에이전트들이 이미 A2A로 대화하고 MCP로 도구를 쓴다면, 여기에 AP2 맨데이트를 추가하는 것만으로 결제 능력을 갖추게 된다.

3

AP2 × x402 — 경쟁이 아닌 협력, A2A x402 확장

AP2와 x402는 다른 문제를 푼다. 그래서 Google과 Coinbase는 경쟁 대신 협력을 택했다.

Coinbase 엔지니어링 헤드 Erik Reppel (2025.09)

"x402와 AP2는 에이전트 간 결제가 더 이상 실험이 아님을 보여줍니다. 개발자들이 실제로 구축하는 방식의 일부가 되고 있습니다. x402를 AP2에 가져와 스테이블코인 결제를 지원하는 것은 자연스러운 일이었습니다."

2025년 9월 AP2 발표와 함께 Google은 Coinbase, Ethereum Foundation, MetaMask와 공동으로 A2A x402 확장을 출시했다. AP2의 맨데이트 시스템 위에서 x402의 스테이블코인 결제가 작동하는 구조다.

AP2가 제공하는 것

  • • 사용자가 권한을 줬다는 암호학적 증명
  • • 에이전트 신원 검증
  • • 분쟁 시 감사 추적
  • • 다양한 결제 방식 지원 (카드, 은행, 스테이블코인)

x402가 제공하는 것

  • • HTTP 레벨의 실제 결제 전송
  • • 계정 없는 즉시 결제
  • • 마이크로트랜잭션 효율성
  • • 개발자 친화적 한 줄 구현

AP2 맨데이트가 "이 에이전트가 $50 이하로 날씨 데이터를 구매할 수 있다"고 증명하면, x402가 실제 API 호출마다 $0.10씩 USDC를 송금한다. 신뢰 레이어와 결제 레이어가 분리된 채로 협력한다.

4

'프로토콜 전쟁'의 진실 — 전쟁이 아니라 스택이다

2026년 초 90일 이내에 모든 주요 결제 플랫폼이 에이전트 결제 표준을 발표했다. 언론은 이를 프로토콜 전쟁으로 부른다. 하지만 실제 구조는 다르다.

4.1 네 가지 표준 비교

Google AP2 신뢰·의도 레이어

60+ 파트너, 맨데이트 시스템, A2A+MCP 통합. 카드·은행·스테이블코인 등 결제 방식 중립. 복잡도는 높지만 법인·기업 거래에 적합. Mastercard, PayPal, American Express 참여.

Coinbase x402 결제 전송 레이어

HTTP 네이티브, 마이크로트랜잭션 최적화, 개발자 친화적. 스테이블코인(USDC) 중심. 1억 건+ 처리. 신원 검증 없음 — AP2 위에서 작동할 때 완전해짐.

Visa TAP 카드 신뢰 레이어

기존 카드 네트워크에 에이전트 신원을 얹는 접근. Agent Identity Certificate로 에이전트 권한 증명. Adyen, Stripe, Nuvei 통합. 카드 수수료·정산 지연 등 기존 레일의 한계 계승.

Stripe ACP / PayPal Agent Ready 실행·분쟁 레이어

어느 프로토콜이 이기든 실행 레이어가 되겠다는 전략. 사기 탐지, 구매자 보호, 분쟁 해결을 에이전트 거래에 추가. AP2·TAP·x402 모두와 통합 가능.

4.2 실제 구조: 레이어드 스택

이 표준들은 상호 배타적이지 않다. AP2의 맨데이트 시스템이 x402 결제 지시를 담을 수 있다. Visa TAP이 카드 네트워크나 스테이블코인 레일로 정산을 승인할 수 있다. PayPal은 어느 프로토콜과도 통합된다. Google과 Coinbase가 A2A x402 확장을 공동 출시한 것이 그 증거다.

에이전트 결제 레이어드 스택

신뢰 레이어
Visa TAP 또는 AP2 맨데이트 — 에이전트 신원·권한 검증
의도 레이어
AP2 맨데이트 — 사용자 지시·조건·한도 기록
결제 레이어
x402 (스테이블코인) 또는 카드 네트워크 (법정화폐)
실행 레이어
Stripe ACP / PayPal Agent Ready — 분쟁·환불·보호

Visa는 신뢰 레이어가 가장 가치 있는 자리라고 본다. Google은 의도 레이어가 핵심이라고 본다. Coinbase는 결제 레이어가 지배적이 될 것이라 본다. PayPal은 실행 레이어가 어느 위에도 작동한다고 본다. 전쟁이 아니라 각자 가장 중요한 레이어를 선점하려는 경쟁이다.

5

한국 시장의 위치 — AP2가 가져오는 기회

AP2의 60개 파트너 목록에 한국 기업은 보이지 않는다. Ant International(알리페이 모체)과 UnionPay International은 있지만, 이들은 중국 결제 생태계를 대표한다. 그러나 이것이 한국과 무관하다는 의미는 아니다.

5.1 카카오페이의 전략적 포지션

카카오페이는 x402의 Linux Foundation 창립 멤버다. AP2는 x402와 공식 협력 관계에 있다. 즉, 카카오페이가 x402를 통해 AP2 생태계에 간접 연결된다. 원화 스테이블코인이 x402 퍼실리테이터로 추가되면, AP2 위에서 원화 결제가 처리될 수 있는 경로가 열린다.

5.2 B2B 에이전트 커머스의 기회

Gartner는 2028년까지 B2B 구매의 90%가 AI 에이전트를 통해 중개될 것이라 예측한다. 이 시장 규모는 15조 달러를 초과한다. AP2가 이 B2B 에이전트 커머스의 신뢰 표준이 된다면, 한국 기업들이 글로벌 AP2 생태계에 접근하기 위한 준비가 필요하다.

  • • 한국 SaaS·데이터 API 기업이 AP2 호환 엔드포인트를 추가하면 글로벌 AI 에이전트의 자동 구매 대상이 됨
  • • Google Cloud Marketplace에서 AP2를 통한 에이전트 자동 소프트웨어 라이선스 구매가 가능해짐
  • • 디지털자산기본법 완성 후 원화 스테이블코인이 x402 퍼실리테이터로 통합되면 AP2 생태계 직접 참여 가능
6

DataClinic이 AP2 경제에서 맡을 자리

AP2가 에이전트 커머스의 신뢰 레이어가 된다면, DataClinic은 어떻게 이 생태계에 위치할 수 있는가.

6.1 위임 구매 시나리오

AP2의 "위임 구매 모드"는 DataClinic에 직접적인 기회를 만든다. 고객사의 데이터 엔지니어가 이렇게 설정할 수 있다.

"이번 주 추가된 데이터셋에 대해 DataClinic 진단을 자동으로 실행하고,
품질 점수가 70점 미만이면 정제 서비스도 함께 구매해줘.
건당 예산 상한 $20, 월 총합 $500."

이 Intent Mandate를 서명하면, 에이전트가 조건을 만족하는 DataClinic API 호출마다 자동으로 결제한다. 사람이 매번 승인하지 않아도 된다. 그러나 맨데이트가 있으므로 분쟁 시 명확한 감사 추적이 존재한다.

6.2 에이전트 마켓플레이스 등록

AP2는 Google AI Agent Marketplace와 연동된다. A2A 에이전트들이 이미 A2A를 통해 서로를 발견하고, AP2를 통해 서로에게 결제한다. DataClinic이 A2A 호환 에이전트로 등록되면 이 생태계에서 자동으로 발견 가능한 유료 서비스가 된다.

6.3 완전한 스택 시나리오

①

고객사 에이전트가 AP2 Intent Mandate 서명 → DataClinic 진단 구매 권한 위임

②

에이전트가 DataClinic MCP 서버에 진단 요청 → x402로 USDC 결제 또는 카드 결제

③

AP2 Cart Mandate로 거래 기록 → 분쟁 시 감사 추적 제공, Stripe/PayPal로 분쟁 처리

DataClinic 연결

DataClinic은 AI·ML 데이터셋의 품질 진단 서비스입니다. AP2의 위임 구매 모드와 x402의 pay-per-query가 결합되면, DataClinic은 기업 데이터 파이프라인에 자동 삽입되는 품질 게이팅 서비스로 진화할 수 있습니다.

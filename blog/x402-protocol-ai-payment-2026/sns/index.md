## Facebook

💡 30년 동안 쓰이지 않은 HTTP 상태 코드가 있었습니다.

"402 — Payment Required."

인터넷이 언젠가 유료화될 것이라 생각한 설계자들이 1991년에 예약해 두었습니다.
신용카드와 구독 모델이 지배하면서 그 코드는 잊혔습니다.

그런데 지난 4월 2일, 이 코드가 Linux Foundation의 공식 표준이 됐습니다.
Google, AWS, Stripe, Visa, Mastercard.
그리고 한국에서는 KakaoPay가 창립 멤버로 이름을 올렸습니다.

x402라는 이름의 이 표준은 간단한 원리입니다.
AI 에이전트가 API를 호출하면 서버가 402로 응답합니다.
에이전트는 자동으로 USDC를 송금하고 데이터를 받습니다.
계정 가입도, API 키 관리도, 사람의 클릭도 없이이죠.

로봇 개 한 마리가 배터리가 부족해지자 스스로 충전소를 찾아가 전기요금을 낸 일도 있었습니다.
x402로이죠.

인터넷의 결제가 사람이 아닌 기계를 위해 설계되기 시작했습니다.
데이터를 파는 비즈니스는 이 변화를 어떻게 활용할 수 있을까요?

## LinkedIn

🚀 HTTP 402가 30년 만에 실제로 작동하기 시작했습니다.

지난 4월 2일, x402 프로토콜이 Linux Foundation에 편입됐습니다.
창립 멤버는 Google, Microsoft, AWS, Stripe, Visa, Mastercard, KakaoPay 등 20개 이상입니다.

x402의 원리는 하나입니다.
서버가 유료 요청에 402로 응답 → 클라이언트(에이전트)가 USDC 자동 송금 → 접근 완료.
계정 없이, API 키 없이, 사람 없이.

6개월 만에 1억 건 이상의 결제가 처리됐습니다.

데이터 비즈니스에 직접적인 시사점이 있습니다.

▸ 구독 모델이 쿼리당 결제로 이동합니다. AI 에이전트는 필요한 만큼만 삽니다.
▸ MCP 서버의 각 도구에 가격표를 붙일 수 있습니다. 코드 한 줄이면 됩니다.
▸ Cloudflare의 pay-per-crawl, Vercel의 x402-next 미들웨어가 이미 작동 중입니다.
▸ KakaoPay가 창립 멤버인 만큼, 원화 스테이블코인 연동 시 한국 서비스 통합 경로도 열립니다.

DataClinic 같은 데이터 품질 진단 서비스가 AI 에이전트 파이프라인에 자동 삽입되는 미래.
그 인프라가 지금 표준화되고 있습니다.

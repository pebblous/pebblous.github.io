사회안전 분야 LLM 파인튜닝용 QA 데이터셋 구축

작성일: 2025년 11월 30일 | 기획: 페블러스 데이터 커뮤니케이션 팀 | 읽는 시간: 약 15분
공유하기:

Ⅰ. 서론 및 구축 목표

이 보고서는 Agentic AI Data Scientist (AADS) 과제에서 대규모 언어 모델(LLM)의 사회안전 도메인 전문성 강화를 목표로, 제공된 데이터셋 문서(데이터 설명서 및 활용 가이드라인)를 기반으로 구축한 질의응답(QA) 쌍 샘플을 요약합니다.

QA 쌍은 다음 4가지 유형으로 분류되었습니다:

A. 도메인 정의/목적

데이터의 최종 목적, 비즈니스 목표, 도메인 정의

B. 데이터 구조/구성

데이터 규모, 포맷, 라벨링 구성 요소 및 분포

C. AI 모델/임무

적용 알고리즘, AI Task 정의, 예측 목표 및 성능 지표

D. 품질/공정 관리

데이터 획득/가공/검수 절차 및 품질 관리 기준

Ⅱ. QA 데이터셋 개요

8

사회안전 데이터셋

32

QA 쌍 (데이터셋당 4쌍)

25%

각 유형 균등 배분

총 8개의 사회안전 도메인 데이터셋에 대해 32개의 QA 쌍을 구축했습니다. 각 데이터셋은 기반시설 안전, 환경 모니터링, 위험 탐지 등 사회안전 분야의 다양한 도메인을 포괄합니다.

QA 쌍은 도메인 정의/목적 (A), 데이터 구조/구성 (B), AI 모델/임무 (C), 품질/공정 관리 (D)의 4가지 유형에 각각 25%씩 균등하게 배분되어, LLM이 사회안전 데이터 과학의 전 영역에 걸친 종합적인 지식을 학습하도록 설계되었습니다.

Ⅲ. 8개 사회안전 도메인 데이터셋

각 데이터셋은 사회안전 분야의 특정 문제를 해결하기 위해 구축되었으며, 멀티모달 데이터(이미지, 센서, 텍스트, 초분광 영상 등)와 고품질 라벨링을 포함합니다.

01. 기반암 시추 시료를 이용한 암반 등급 분류 데이터

🎯 도메인 정의/목적

암반 등급 판정의 정확도와 객관성을 높여 터널 및 비탈면 공사 붕괴 사고 예방

📊 데이터 구조/구성

암종 분류: 550,080쌍 (이미지 + JSON)
절리 탐지: 111,345쌍 (이미지 + JSON)

🤖 AI 모델/임무

암종 분류: ResNet (Image Classification)
절리 탐지: Deeplab V3 (Segmentation)

✅ 품질/공정 관리

Hashlib 라이브러리로 해시값 비교하여 중복 이미지 제거

02. 지능형 관제 서비스 CCTV 영상 데이터

🎯 도메인 정의/목적

침입, 싸움, 쓰러짐, 군집, 인파밀집, 침수 등 6종 안전사고 자동 탐지로 조기 감지 및 신속 대응

📊 데이터 구조/구성

총 300건 영상
안전사고 4종(200건) + 특화 이벤트 2종(100건)

🤖 AI 모델/임무

임무: 이상 행위 탐지
성능 목표: AUC 80% (불균형 데이터셋 대응)

✅ 품질/공정 관리

개인정보 비식별화(블러링/마스킹)로 편향 제거 및 일반화 능력 향상

03. 건설용 자갈 품질관리 데이터

🎯 도메인 정의/목적

콘크리트 품질에 직접 영향을 미치는 자갈 품질을 AI 자동 분석하여 불량 골재 유통 차단

📊 데이터 구조/구성

자갈 암석 종류: Polygon Segmentation
편장석 비율: Rotated Bounding Box

🤖 AI 모델/임무

YOLO-OBB (Oriented Bounding Box)
성능: IoU@50, mAP 75%

✅ 품질/공정 관리

Dup Detector(중복 제거) + Sony image edge view(육안 전수 검사)

04. 자연발생 석면 탐지 데이터

🎯 도메인 정의/목적

자연 환경 석면 및 석면 함유 암석을 신속·정확 탐지하여 건강 위협 사전 관리

📊 데이터 구조/구성

멀티모달: 광학(JPG) + VNIR 초분광(TIF) + SWIR 초분광(TIF)
3가지 파장대역 융합

🤖 AI 모델/임무

Early Fusion vs Late Fusion 비교 평가
멀티모달 통합 전략 최적화

✅ 품질/공정 관리

반사도/대기/기하 보정으로 물질 고유 분광 특성 복원

05. SOC 시설물 균열패턴 이미지 데이터

🎯 도메인 정의/목적

교량·터널 등 위험 지역 시설물 균열 자동 진단 AI 서비스 및 탐지 로봇 개발

📊 데이터 구조/구성

10가지 클래스: 균열, 망상균열, 박리, 박락, 백태, 누수,
철근노출, 재료분리, 들뜸, 파손

🤖 AI 모델/임무

분류: CvT (Convolutional Vision Transformer)
분할: SegFormer

✅ 품질/공정 관리

SSIM 0.9 이하 프레임 추출로 과적합 방지 및 다양성 확보

06. 놀이기구 및 시설 이용자 위험 상황 인식 데이터

🎯 도메인 정의/목적

놀이기구 및 시설의 위험 상황과 파손 상태 자동 인식으로 사고 예측·경고 및 즉각 대응

📊 데이터 구조/구성

3가지 라벨링: 바운딩박스(객체 탐지) + 키포인트(자세 분석) +
세그멘테이션(파손 영역)

🤖 AI 모델/임무

InternImage-L Foundation 모델 기반
전이 학습으로 효율적 미세조정

✅ 품질/공정 관리

4단계 절차: 가공 → 검수 → 재가공 → 검증 (피드백 루프)

07. 내륙습지 탄소흡수원 데이터

🎯 도메인 정의/목적

내륙습지 식생·수역 분석으로 탄소 흡수량 추정 및 변화 모니터링 (기후 변화 대응)

📊 데이터 구조/구성

멀티소스: Sentinel 위성영상 + 드론영상 +
수치자료(강수량, 기온, 수위)

🤖 AI 모델/임무

드론: Trans UNet
위성: Modified Trans UNet (데이터 특성 최적화)

✅ 품질/공정 관리

SNAP 소프트웨어로 Sentinel-1 표준 전처리 (방사/지형 보정)

08. 화학물질 위험성 예측 데이터

🎯 도메인 정의/목적

물리화학적 특성으로 미지 화학물질 위험성 예측, REACH 규제 대응 및 안전 기준 수립

📊 데이터 구조/구성

3가지 특성: 증기압(5,110건) + 연소열(5,005건) +
인화점(10,010건)

🤖 AI 모델/임무

웹 기반 예측 서비스
분자 구조 입력 → 즉시 위험성 예측 (접근성 극대화)

✅ 품질/공정 관리

GHS 정보를 '위험/자료없음/해당없음'으로 표준화하여 정합성 확보

Ⅳ. 질의-응답 유형 최종 통계

LLM 학습 데이터 생성을 위해 총 8개의 사회안전 도메인 데이터셋에 대해 32개의 질의응답 쌍을 구성했습니다. 데이터 과학의 사회안전 도메인 적용 측면을 반영한 질의 유형 통계는 다음과 같습니다.

질의 유형 정의 사용 횟수 비율
A. 도메인 정의/목적 데이터의 최종 목적, 비즈니스 목표, 도메인 정의 8회 25.0%
B. 데이터 구조/구성 데이터 규모, 포맷, 라벨링 구성 요소 및 분포 등 8회 25.0%
C. AI 모델/임무 적용 알고리즘, AI Task 정의, 예측 목표 및 성능 지표 8회 25.0%
D. 품질/공정 관리 데이터 획득/가공/검수 절차 및 품질 관리 기준 8회 25.0%
총합 32회 100.0%

💡 핵심 특징

제조 도메인 데이터 과학의 핵심 요소인 목적 (A), 데이터 특징 (B), 기술 적용 (C), 품질 관리 (D) 영역에 대해 균등하게 질문을 배분하여 LLM이 전 영역에 걸친 종합적인 지식을 학습하도록 설계되었습니다.

Ⅴ. 도메인 LLM 보고서 생성을 위한 프롬프트 템플릿

이 프롬프트는 다른 도메인(예: 로봇, 헬스케어, 자율주행 등)의 학습 데이터 문서가 주어졌을 때, 해당 도메인의 전문 지식을 LLM이 학습할 수 있도록 구조화된 QA 데이터셋 보고서를 생성하는 데 사용될 수 있습니다.

Report Generation Prompt Template

[지시사항]
당신은 Agentic AI Data Scientist (AADS) 과제에서 대규모 언어 모델(LLM) 파인튜닝을 위한
전문 QA 데이터셋을 구축하는 전문가입니다. 아래에 제시된 [INPUT: 분석 대상 문서]의 내용을 분석하여,
**'논리적 데이터 그룹'** 단위로 묶어 QA 보고서를 생성해야 합니다.

**[보고서 구성 요소]**
1. **보고서 제목:** 도메인 및 목적에 맞게 작성하십시오.
2. **논리적 데이터 그룹 식별:** 문서 내에서 동일한 프로젝트나 목표를 공유하는 문서들을
   하나의 '논리적 그룹'으로 묶습니다.
3. **QA 쌍 생성:** 각 논리적 그룹별로 **2개**의 질의응답(QA) 쌍을 생성해야 합니다.
4. **질의 유형 분류:** 생성된 QA 쌍은 다음 4가지 핵심 유형 중 하나로 분류되어야 합니다.
   * **A. 도메인 정의/목적:** 해당 데이터가 해결하려는 산업 문제와 비즈니스 목적
   * **B. 데이터 구조/구성:** 데이터의 규모, 포맷, 라벨링 구성 요소 및 분포
   * **C. AI 모델/임무:** 적용 알고리즘, AI 임무 정의, 예측 목표, 성능 지표
   * **D. 품질/공정 관리:** 데이터 획득/가공/검수 절차, 라벨링 기준, 품질 관리 기준
5. **출처 표기:** 응답의 모든 문장은 [i] 형식으로 원본 문서의 출처를 명확하게 표기해야 합니다.
6. **최종 통계:** 생성된 모든 QA 쌍을 대상으로, 사용된 **A, B, C, D 유형의 최종 횟수와 비율**을
   정리해야 합니다.

활용 방법: 이 템플릿을 사용하여 다양한 도메인(헬스케어, 자율주행, 로봇 등)의 학습 데이터 문서에서 고품질 QA 데이터셋을 자동으로 생성할 수 있습니다.

페블러스 관점: 사회안전 AI 시대의 데이터 중심 접근법

이번 AADS 과제에서 구축한 사회안전 분야 QA 데이터셋은 Safety AI 시대에 데이터 품질이 만드는 지능의 차이를 명확히 보여줍니다.

🎯 AADS의 차별화된 접근법

  • •
    균형잡힌 지식 구조: 도메인 정의, 데이터 구조, AI 모델, 품질 관리 영역을 25%씩 균등 배분하여 LLM이 편향되지 않은 전문성을 습득하도록 설계
  • •
    실무 중심 QA: 8개 사회안전 도메인의 실제 데이터셋 문서에서 추출한 사실 기반(Factual) 질의응답으로 환각(Hallucination) 최소화
  • •
    확장 가능한 템플릿: 프롬프트 템플릿을 통해 다른 도메인(제조, 헬스케어 등)으로 즉시 확장 가능한 재사용 가능 구조

DataClinic과의 연계

AADS가 구축한 고품질 QA 데이터셋은 DataClinic의 데이터 품질 진단 및 정제 파이프라인과 결합하여, LLM 파인튜닝 전 데이터 품질을 사전 검증하고 최적화합니다.

피지컬 AI 전략

제조 현장의 멀티모달 데이터(센서, 이미지, 텍스트)를 LLM이 이해하도록 파인튜닝함으로써, 물리적 세계와 상호작용하는 AI의 기반을 마련합니다.

페블러스는 이번 과제를 통해 데이터 중심 AI (Data-Centric AI) 접근법의 중요성을 재확인했습니다. 단순히 모델 크기를 키우는 것이 아니라, 도메인 전문성이 담긴 고품질 데이터로 파인튜닝할 때 진정한 지능이 창발한다는 것을 보여주는 사례입니다.

자주 묻는 질문 (FAQ)

AADS가 사회안전 분야 QA 데이터셋을 어떻게 구축하나요?

AADS(Agentic AI Data Scientist)는 페블러스가 개발한 데이터 중심 AI 에이전트로, 사회안전 분야의 실제 데이터셋 문서(데이터 설명서, 활용 가이드라인)를 체계적으로 분석하여 8개 핵심 도메인을 선정했습니다. AADS는 각 데이터셋별로 4가지 유형(도메인 정의, 데이터 구조, AI 모델, 품질 관리)에서 각 1개씩 QA 쌍을 생성하여 총 32개의 고품질 질의응답을 구축했습니다. 이는 단순한 데이터 수집이 아니라, 안전 도메인의 전문 지식과 품질 관리 논리를 LLM이 체계적으로 학습할 수 있도록 설계된 지능형 접근 방식입니다.

사회안전 분야에서 데이터 품질 관리가 왜 특히 중요한가요?

사회안전 분야는 사람의 생명과 직결되기 때문에 데이터 품질이 다른 어떤 분야보다 중요합니다. 석면 탐지 오류는 작업자의 건강 피해로 이어지고, SOC 시설물 균열 오진단은 교량·터널 붕괴 사고를 초래하며, 놀이기구 안전 미탐은 이용자 사망 사고로 직결됩니다. AADS는 초분광 영상의 대기 보정, CCTV 영상의 개인정보 비식별화, 균열 패턴의 SSIM 기반 중복 제거 등 생명안전을 위한 품질 관리 절차를 학습하여, 안전 AI의 신뢰성을 확보합니다. 제조업 데이터의 품질 오류가 제품 불량으로 끝나는 것과 달리, 사회안전 데이터의 품질 오류는 회복 불가능한 인명 피해를 야기합니다.

8개 데이터셋은 어떤 안전 영역을 다루나요?

기반시설 안전(기반암 시추, SOC 시설물 균열), 공공 안전(지능형 관제 CCTV, 놀이기구 위험 상황), 건설 안전(건설용 자갈 품질), 환경 안전(석면 탐지, 내륙습지 탄소흡수원), 화학 안전(화학물질 위험성 예측) 등 사회안전의 핵심 영역을 포괄합니다. 이를 통해 LLM이 다양한 안전 도메인의 전문 지식을 균형있게 학습합니다.

멀티모달 데이터가 사회안전 AI에 어떤 이점을 제공하나요?

사회안전 분야는 다양한 센서와 데이터 소스를 결합해야 정확한 판단이 가능합니다. 예를 들어 석면 탐지 데이터는 광학 이미지 + VNIR 초분광 영상 + SWIR 초분광 영상을 융합하여, LLM이 '석면은 특정 파장대에서만 식별 가능하므로 멀티모달 Early Fusion이 효과적'이라는 지식을 학습합니다. 내륙습지 데이터는 위성영상 + 드론영상 + 기상 수치자료를 통합하여 탄소흡수량을 정확히 추정합니다.

데이터 품질 관리(D 유형)가 안전 AI에서 더 중요한 이유는?

안전 AI는 오탐(False Positive)과 미탐(False Negative)이 모두 치명적입니다. AADS는 '지능형 관제 CCTV의 AUC 80% 목표는 불균형 데이터셋(정상 99%, 이상 1%)에서 이상 행위를 강건하게 탐지하기 위함', '놀이기구 안전 데이터의 4단계 품질 관리(가공→검수→재가공→검증)는 신체 키포인트 정확도를 보장하여 추락 위험을 사전 감지하기 위함', 'SOC 시설물 균열 데이터의 SSIM 0.9 이하 프레임 추출은 유사한 균열 이미지 중복을 방지하여 모델 과적합을 막기 위함'과 같은 생명안전 중심의 품질 관리 논리를 학습합니다. 이를 통해 안전 AI 개발 시 도메인 특성에 맞는 성능 지표와 검증 절차를 제안할 수 있습니다.

프롬프트 템플릿을 다른 안전 도메인에 적용할 수 있나요?

네, 프롬프트 템플릿은 확장 가능하도록 설계되었습니다. 사회안전의 4가지 질의 유형(A: 도메인 정의, B: 데이터 구조, C: AI 모델, D: 품질 관리)은 산업 안전(제조 현장 위험 감지), 교통 안전(자율주행 위험 예측), 식품 안전(유해물질 탐지) 등으로 확장할 수 있습니다. 핵심은 각 도메인의 안전 기준과 규제 요구사항을 D 유형 질의에 정확히 반영하는 것입니다.

AADS 사회안전 QA 데이터셋이 DataClinic과 어떻게 연계되나요?

DataClinic(데이터클리닉)은 페블러스의 데이터 품질 진단 시스템으로, 사회안전 데이터셋의 품질 이슈를 자동으로 탐지합니다. 예를 들어 DataClinic이 'SOC 시설물 균열 데이터에서 라벨링 불일치 87건 발견', '초분광 영상의 특정 밴드에서 SNR 3.0 미만 노이즈 검출', 'CCTV 영상의 야간 프레임 조도 불균형(표준편차 15 이상)' 같은 품질 문제를 진단하면, AADS는 해당 문제에 대한 QA 쌍을 학습하여 'SOC 시설물 균열 데이터의 라벨 불일치는 SSIM 0.9 이하 프레임 추출로 해결', '초분광 영상은 대기 보정 + 밴드 정규화 필수', 'CCTV 야간 영상은 Adaptive Histogram Equalization 적용' 같은 도메인 특화 해결책을 제시합니다. 이는 DataClinic(품질 탐지) → AADS(해결책 학습) → 안전 데이터 신뢰성 향상으로 이어지는 선순환 구조를 만들어, 사회안전 AI의 실전 배포 가능성을 높입니다.

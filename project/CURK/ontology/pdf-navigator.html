Why

왜 만들었나요?

뉴로-심볼릭 AI로의 여정

벡터 임베딩 + RAG에서 시작해, 온톨로지와 지식그래프의 결합은 필연입니다. 2000년대 초 ETRI에서 배운 OWL과 Topic Map, 그리고 현재 페블러스 데이터클리닉의 Agentic AI 개발 경험이 만나 하이브리드 AI를 향합니다.

ISO 5259-2를 동적 품질 관리로

AI/ML 데이터 품질 표준을 실용적 온톨로지로 구현하고, 품질 기준-데이터-개선 방법을 동적 루프에서 관리하는 것이 목표입니다. 팔란티어 온톨로지처럼 액션의 결과를 학습하고 지식으로 축적하는 시스템으로 발전시킵니다.

What

무엇인가요?

온톨로지 기반 PDF 탐색 도구

  • 용어 정의 (3장 핵심 개념)
  • ISO 메타 온톨로지

📚 활용 가이드

온톨로지 기반 PDF 탐색기를 다양한 시나리오에서 활용하는 방법을 소개합니다.

🎯

1. 표준 문서 학습

ISO/IEC 5259-2 데이터 품질 표준을 체계적으로 학습할 때, 온톨로지 그래프로 전체 구조를 파악하고 각 개념이 문서 어디에 정의되어 있는지 빠르게 찾을 수 있습니다.

🔍

2. 개념 간 관계 탐색

데이터 품질 특성(Accuracy, Completeness 등)이 서로 어떻게 연결되어 있는지 그래프 뷰에서 시각적으로 확인하고, 관련 정의를 PDF에서 즉시 참조할 수 있습니다.

📝

3. 기술 문서 작성

데이터 품질 관련 제안서나 보고서를 작성할 때, 특정 품질 특성(예: Credibility)의 모든 언급을 한눈에 보고 정확한 정의와 맥락을 인용할 수 있습니다.

🎓

4. 교육 및 트레이닝

데이터 품질 교육 과정에서 강사가 온톨로지 그래프를 보여주며 개념을 설명하고, 학습자가 PDF에서 직접 해당 섹션을 찾아가며 학습할 수 있습니다.

🏗️

5. 데이터 품질 프레임워크 설계

조직의 데이터 품질 관리 프레임워크를 구축할 때, 표준에 정의된 품질 특성 계층 구조를 참고하고 각 특성의 정의를 확인할 수 있습니다.

🤖

6. AI 시스템 학습 데이터 준비

데이터 품질 관련 AI 모델을 학습시킬 때, 온톨로지 구조와 표준 문서의 정확한 매핑을 활용하여 고품질 학습 데이터를 생성할 수 있습니다.

✅

7. 표준 준수성 검증

기존 데이터 품질 시스템이 ISO 표준을 준수하는지 검증할 때, 각 품질 지표가 표준의 어떤 개념과 매핑되는지 확인할 수 있습니다.

📊

8. 데이터 품질 평가 설계

데이터 품질 평가 체크리스트를 만들 때, 트리 뷰에서 모든 품질 특성을 순회하며 각 항목에 대한 평가 기준을 표준 문서에서 추출할 수 있습니다.

🔗

9. 다른 표준과의 매핑

ISO/IEC 25012 또는 다른 데이터 품질 표준과 ISO 5259-2를 비교할 때, 온톨로지 구조를 활용하여 개념 간 대응 관계를 분석할 수 있습니다.

🚀

10. 표준 문서 온톨로지화 사례 연구

다른 ISO 표준이나 기술 문서를 온톨로지로 변환하는 프로젝트의 레퍼런스로 활용하여, 구조화된 지식 표현 방법론을 학습할 수 있습니다.

🧩 온톨로지 레이어 이해하기

이 탐색기는 4가지 온톨로지 레이어를 제공합니다. 각 레이어는 문서를 서로 다른 관점에서 바라봅니다:

📊 품질 특성 온톨로지

데이터 품질 평가 항목의 계층 구조를 나타냅니다. Accuracy, Completeness 등의 품질 특성이 고유 특성(Inherent)과 시스템 의존 특성(System-dependent)으로 분류되는 구조를 시각화합니다.

📖 문서 구조 온톨로지

ISO 표준 문서의 물리적 구조를 나타냅니다. 섹션(1, 2, 3...), 하위 조항(6.2.1, 6.2.1.1...), 부속서(Annex A, B...)의 계층 관계를 보여주며, 각 섹션이 PDF의 어느 페이지에 위치하는지 연결합니다.

💡 용어 정의 온톨로지

문서 3장(Terms and definitions)에 정의된 핵심 용어들을 추출한 것입니다. "data value", "empty data item", "quality measure element" 등 이 표준에서 사용하는 도메인 특화 개념들을 나열하며, 각 용어의 정의 위치를 PDF에서 바로 확인할 수 있습니다.

🏛️ ISO 메타 개념 온톨로지

모든 ISO/IEC 표준에 공통으로 적용 가능한 메타 개념을 정의합니다. Section, Definition, Requirement, Example, Note 등 표준 문서의 구조와 내용 유형을 OWL/RDF로 표현하며, 이 문서뿐만 아니라 다른 ISO 표준에도 재사용할 수 있는 범용 온톨로지입니다. TTL 파일 보기 →

🔍 핵심 포인트: 위 4개 레이어는 서로 다른 시점(viewpoint)을 제공합니다. 품질 특성은 도메인 지식, 문서 구조는 물리적 조직, 용어 정의는 이 문서의 개념, ISO 메타 개념은 모든 표준에 공통되는 구조를 나타냅니다.

⚙️ 온톨로지 추출 및 구축 방법론

1. 품질 특성 온톨로지

추출 방법: 기존 OWL 온톨로지 파일 ([iso5259-2-ontology.owl](example/iso5259-2-ontology.owl)) 기반

  • • 도메인 전문가가 ISO/IEC 5259-2 표준을 분석하여 수작업으로 구축
  • • 34개 클래스: DataQualityModel, Accuracy, Completeness 등
  • • PDF 전체 텍스트를 정규표현식으로 스캔하여 857개 언급 위치 인덱싱
  • • 계층 구조: InherentCharacteristic ⊃ {Accuracy, Completeness, ...}
  • • build-all-ontologies.py의 품질 특성 인덱싱 로직 사용

2. 문서 구조 온톨로지

추출 방법: PyPDF2 + 정규표현식 기반 자동 추출

  • • 패턴 매칭: r'^([\d.]+)\s+([A-Z][^\n]{"{5,80}"])$' - 섹션 헤더 탐지
  • • 계층 구조: 섹션 번호를 '.'로 분할하여 부모-자식 관계 자동 생성
  • • 결과: 92개 섹션 (Section 1, 2, 3 → 6.2.1.1, 6.2.1.2, ...)
  • • OWL 트리플: Section_6_2_1 iso:hasSubsection Section_6_2_1_1
  • • 스크립트: build-all-ontologies.py:build_document_structure_ontology()

3. 용어 정의 온톨로지

추출 방법: 정규표현식 기반 간단 추출 (LLM 불필요)

  • • 3장 "Terms and definitions"에서 용어 패턴 탐지
  • • 패턴: r'^(\d+\.\d+)\s*\n\s*([a-z][a-z\s-]+)$'
  • • 결과: 19개 용어 (data value, empty data item, quality measure element 등)
  • • 각 용어의 정의 위치(페이지, 섹션)를 PDF와 연결
  • • 스크립트: extract-document-structure.py

4. ISO 메타 개념 온톨로지

구축 방법: 수작업 OWL/TTL 작성 (범용 재사용 목적)

  • • 참고문헌: ISO/IEC Directives, Part 2 (표준 문서 구조 규칙)
  • • 정의된 클래스: Section, Clause, Annex, Definition, Requirement, Example, Note 등 16개
  • • 카테고리: 문서 구조 개념(4), 내용 유형 개념(3), 메타 개념(9)
  • • 재사용성: 이 표준뿐 아니라 모든 ISO/IEC 표준에 적용 가능
  • • 파일: iso-standard-meta-ontology.ttl → JSON 변환: convert-meta-ontology-to-index.py

📚 학술적 근거: ISO 메타 온톨로지는 ISO/IEC Directives, Part 2: Principles and rules for the structure and drafting of ISO and IEC documents 를 기반으로 합니다. 이 문서는 Clause 6(구조), Clause 7(표현 방식), Clause 14-21(문서 요소)에서 표준 문서의 공통 구조를 명시하며, 모든 ISO/IEC 표준이 따라야 하는 규칙을 정의합니다.

  • ▸ 그래프 뷰에서 노드를 클릭하면 해당 클래스의 모든 언급 위치가 "문서 위치" 탭에 표시됩니다.
  • ▸ 트리 뷰의 "모두 열기/닫기" 버튼으로 전체 계층 구조를 한눈에 보거나 접을 수 있습니다.
  • ▸ PDF에서 텍스트를 드래그하면 자동으로 매칭되는 온톨로지 클래스를 찾아줍니다.
  • ▸ 문서 위치 카드를 클릭하면 PDF의 해당 페이지로 이동하며 키워드가 하이라이트됩니다.
  • ▸ 데스크톱에서는 두 패널 사이의 구분선을 드래그하여 온톨로지 뷰와 PDF 뷰의 비율을 조정할 수 있습니다.
  • ▸ 모바일에서는 왼쪽의 햄버거 메뉴로 온톨로지 패널을 열고 닫을 수 있습니다.
